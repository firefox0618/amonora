# 📚 Глоссарий Amonora

Алфавитный справочник всех терминов, используемых в проекте.

---

## А

### Аккаунт (Account)
Учётная запись пользователя в системе Amonora. Создаётся автоматически при первом запуске бота (`/start`). Привязан к Telegram ID.

### Активация (Activation)
Процесс привязки устройства к VPN-ключу. Происходит при первом подключении клиента к серверу. Фиксируется в таблице `vpn_client_activations`.

### Алерт (Alert)
Уведомление о критическом событии в системе. Отправляется в [@amonora_control_bot](https://t.me/amonora_control_bot). Примеры: падение ноды, ошибка платежа, исчерпание лимитов.

### Антишаринг (Anti-sharing)
Механизм защиты от передачи доступа третьим лицам. Ограничивает количество одновременных подключений по одному ключу.

**Параметры:**
- `VPN_ANTISHARING_LEASE_SECONDS=180` — время жизни сессии (3 минуты)
- `VPN_ANTISHARING_SOFT_LIMIT_ENABLED=1` — мягкое ограничение включено

## Б

### Баланс (Balance)
Внутренний счёт пользователя в рублях. Используется для оплаты подписок, покупки слотов.

**Типы:**
- **Баланс** (`balance_rub`) — доступные средства
- **Зарезервированный баланс** (`balance_reserved_rub`) — средства в процессе оплаты

**Пополнение:** через реферальную программу, промокоды, ручные зачисления админом.

### Бот (Bot)
Telegram-бот для взаимодействия с пользователями или администраторами.

**Виды:**
- [@amonora_bot](https://t.me/amonora_bot) — клиентский бот (основной)
- [@amonora_support_bot](https://t.me/amonora_support_bot) — бот поддержки
- [@amonora_control_bot](https://t.me/amonora_control_bot) — бот уведомлений для админов
- [@test_amonora_bot](https://t.me/test_amonora_bot) — тестовый бот для отладки

### Bridge-user (Мост)
Служебный пользователь для синхронизации данных между старой и новой системой. Имеет префикс `bridge_` в email VPN-клиента.

---

## В

### WireGuard
Современный VPN-протокол. Используется как резервный вариант для некоторых регионов.

**Параметры:**
- `WIREGUARD_ENDPOINT` — адрес сервера
- `WIREGUARD_DNS` — DNS-серверы (1.1.1.1, 8.8.8.8)

### Воронка (Funnel)
Последовательность шагов, которые проходит пользователь от первого визита до оплаты.

**Этапы:**
1. `/start` в боте
2. Получение trial
3. Выбор страны/протокола
4. Первая оплата
5. Продление

### ВLESS
Основной протокол VPN в Amonora. Современный протокол на базе V2Ray, обходит блокировки.

---

## Д

### Dashboard (Дашборд)
Веб-панель управления для администраторов. Находится по внутреннему адресу (порт 8088).

**Возможности:**
- Управление пользователями и тикетами
- Просмотр аналитики и финансов
- Настройка тарифов и регионов
- Мониторинг нод

**Доступы:**
- Владелец: `rudolf` / `DASHBOARD_OWNER_PASSWORD`
- Технарь: `ruslan` / `DASHBOARD_TECH_PASSWORD`
- Поддержка: `muradym` / `DASHBOARD_SUPPORT_PASSWORD`

### Девайс (Device)
Устройство пользователя (телефон, компьютер, планшет). Каждое устройство требует отдельного слота.

**Лимиты:**
- `VPN_MAX_DEVICES_PER_KEY=1` — базовый лимит (1 устройство)
- Можно купить дополнительные слоты через оплату

### Дедупликация (Deduplication)
Механизм предотвращения дублирования событий (платежей, активаций, аналитики). Использует уникальные ключи (`dedupe_key`).

## И

### Инцидент (Incident)
Критическое событие, требующее вмешательства администратора. Фиксируется в Grafana и контрол-боте.

**Категории:**
- `access` — проблемы с доступом пользователей
- `node` — проблемы с VPN-нодами
- `service` — сбои сервисов (бот, база, платежи)
- `payment` — ошибки платежей

---

## К

### Канал (Channel)
Telegram-канал Amonora. Используется для обязательной подписки перед получением trial.

**Параметр:** `CHANNEL_ID`

### Конфиг (Config)
Конфигурация VPN-подключения. Может быть в формате:
- QR-код
- Ссылка `vless://...`
- Файл `.conf` для WireGuard

### Контрол-бот
См. **Бот** (@amonora_control_bot).

---

## М

### Миграция (Migration)
Изменение структуры базы данных. Выполняется автоматически при старте (`AUTO_APPLY_SCHEMA=1`).

**Файлы:** `backend/core/schema.py`

### Мост (Bridge)
См. **Bridge-user**.

---

## Н

### Ноде (Node)
VPN-сервер в определённой стране.

**Активные ноды:**
- **DE** (Германия) — 3x-ui панель
- **DK** (Дания) — Xray Core

**Устаревшие:**
- **EE** (Эстония) — требует миграции

---

## О

### Орафан (Orphan)
«Бесхозный» объект без привязки к пользователю.

**Типы:**
- **Orphan remote** — VPN-клиент на сервере без записи в БД
- **Orphan local** — запись в БД без клиента на сервере

---

## П

### Платёж (Payment)
Оплата подписки или услуг.

**Методы:**
- **Telegram Stars** (`XTR`) — встроенная валюта Telegram
- **SBP** (СБП) — через Platega
- **Crypto** — USDT, TON через Crypto Pay
- **Manual** — ручное зачисление админом

**Статусы:** `pending`, `paid`, `failed`, `refunded`

### Промокод (Promo Code)
Уникальный код для получения скидки или бонуса.

**Типы:**
- Скидка на первую оплату
- Бесплатные дни
- Бонусы на баланс

**Таблица:** `promo_codes`

### Пробный период (Trial)
Бесплатный доступ на 3 дня для новых пользователей.

**Условия:**
- Даётся один раз на аккаунт
- Требуется подписка на канал
- `TRIAL_DAYS=3`

### Профиль (Profile)
Настройки пользователя: предпочтения протокола, страны, режима работы.

**Поля:**
- `preferred_mode` — режим (stable, mobile)
- `preferred_protocol` — протокол (vless, wireguard)

---

## Р

### Реферал (Referral)
Пользователь, пришедший по приглашению другого пользователя.

**Бонусы:**
- Пригласивший получает 3 месяца бесплатно
- Приглашённый получает скидку

**Таблицы:** `referrals`, `referral_rewards`

### Ремонт (Repair)
Автоматическое восстановление VPN-доступа при проблемах.

**Причины:**
- Истёк срок на сервере
- Клиент удалён с ноды
- Ошибка конфигурации

**Таблица:** `vpn_repair_events`

### Ротация (Rotation)
Периодическая смена токенов, паролей, ключей доступа.

---

## С

### СБП (SBP)
Система быстрых платежей (Россия). Интеграция через Platega.

**Параметр:** `ENABLE_PLATEGA_SBP_USER_FLOW=1`

### Слот (Slot)
Право подключить одно устройство. Базовый тариф = 1 слот.

**Таблица:** `device_slot_entitlements`

### Синтетический пользователь (Synthetic User)
Тестовый аккаунт для автоматических проверок. Имеет флаг `is_synthetic=True`.

---

## Т

### Тариф (Tariff)
План подписки с фиксированной ценой и сроком.

**Коды:** `1m`, `3m`, `6m`, `12m` (месяцы)

**Цены (RUB):**
- `TARIFF_1M_RUB=149`
- `TARIFF_3M_RUB=399`
- `TARIFF_6M_RUB=749`
- `TARIFF_12M_RUB=1390`

### Тикет (Ticket)
Обращение пользователя в поддержку.

**Статусы:** `new`, `in_progress`, `resolved`, `closed`

**Таблицы:** `support_tickets`, `support_ticket_messages`

### Триал (Trial)
См. **Пробный период**.

## У

### Устройство (Device)
См. **Девайс**.

## Ф

### Фingerprint (Отпечаток)
Уникальный идентификатор устройства. Используется для антишаринга.

**Хэш:** SHA-256 от комбинации device_id + user_agent

---

## Х

### Xray Core
Движок для протокола VLESS. Используется на датской ноде (DK).

### 3x-ui
Веб-панель управления VLESS-клиентами. Используется на немецкой ноде (DE).

---

## Э

### Email (VPN)
Уникальный идентификатор VPN-клиента в панели. Формат: `<prefix>_<user_id>@amonora.local`

**Префиксы:**
- `device_` — клиентские устройства
- `dashboard_` — служебные доступы
- `bridge_` — мосты для миграции
- `test_` — тестовые профили
- `owner_`, `admin_` — админские доступы

---

## Ю

### User Agent
Идентификатор приложения/устройства при подключении. Используется для аналитики и антишаринга.
