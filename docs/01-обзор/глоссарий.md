# Глоссарий терминов Amonora

Этот словарь описывает термины, используемые в коде, документации и общении команды. Определения даны с привязкой к реалиям проекта Amonora.

## A

- **Админка (Dashboard)** — Веб-панель управления для персонала. Backend: FastAPI на `127.0.0.1:8088`. Frontend: Next.js 16 на `127.0.0.1:3001`. Администраторы видят пользователей, балансы, платежи, тикеты, настройки тарифов, статус нод. Вход: логин/пароль + 5-минутный Telegram-код через `@amonora_control_bot`. Роли: Владелец / Тех. администратор / Менеджер.

- **Алерт (Alert)** — Тревожное уведомление. Приходит в `@amonora_control_bot`, если что-то сломалось: нода не отвечает, закончилось место на сервере, критическая ошибка в боте. Типизированные события (`ControlNotificationEvent`) хранятся в PostgreSQL.

- **API** — Программный интерфейс между компонентами. В Amonora: `backend` предоставляет общий API для ботов и dashboard. Dashboard предоставляет API для `dashboard/ui`. Platega и Crypto Pay имеют внешние API для оплат.

- **Anti-DPI** — Технологии обхода глубокой инспекции пакетов. Датская нода (`dk.amonoraconnect.com`) использует Xray core с XHTTP для устойчивости к DPI.

## Б

- **Баланс (`UserBalanceEvent`)** — Внутренний рублёвый счёт пользователя. Пополняется за рефералов (50₽ / 100₽ за 12 месяцев). Авто-списывается при оплате подписки. Хранится в PostgreSQL.

- **Бэкенд (Backend)** — Серверная часть проекта: `backend/`. Модели, схема БД, миграции, доменные сущности. Без него не работает ни один сервис. Ядро системы.

- **Бот клиентский** (`@amonora_bot`) — Главный бот для пользователей. Единая подписка, trial, оплата (Stars + Platega), рефералы, баланс, промокоды, переход в поддержку. Основной UI-роутер: `bot/router.py`. Написан на aiogram 3.

- **Бот контрольный** (`@amonora_control_bot`) — Внутренний бот команды. Системные уведомления, review платежей (confirm/reject), 5-минутные коды входа в админку, рассылки, триггеры, node alerts. Только для администраторов.

- **Бот поддержки** (`@amonora_support_bot`) — Бот для общения с клиентами. Приём обращений, создание тикетов, диалоги, медиа (фото/видео/аудио), назначение ответственного, закрытие. **Не получает** auth-коды, payment review, node alerts.

- **Bridge-user (Синтетический пользователь)** — Пользователь, созданный через bridge-access на сайте (`POST /bridge/access`). Получает бесплатный ключ на 24 часа, если Telegram недоступен. Это мост до `@amonora_bot`, не отдельный тариф. Синтетические bridge-users не попадают в обычные агрегаты панели (`synthetic_users.py`).

## В

- **Вебхук (Webhook)** — Механизм мгновенного уведомления внешних сервисов. В Amonora: `POST /webhooks/platega/{secret}` — callback от Platega об успешной оплате. Без вебхуков пришлось бы опрашивать платёжную систему вручную.

- **Воркер (Worker)** — Фоновый процесс, выполняющий рутинную работу: рассылка уведомлений, проверка истёкших подписок, генерация конфигов. В Amonora каждый бот — отдельный systemd-сервис (воркер).

- **Воронка** — Путь пользователя от первого контакта до целевого действия. В Amonora: *Зашел в бот → Нажал «Старт» → Принял соглашение → Подписался на канал → Взял триал → Получил единую ссылку → Купил подписку*. Аналитика воронок хранится в `backend/core/analytics.py`.

## Г

- **Гео (Geo)** — Сокращение от «Геолокация». В контексте Amonora — серверы в разных странах: Германия (`ffconnect.amonoraconnect.com`), Дания (`dk.amonoraconnect.com`), Эстония (`est.amonoraconnect.com`). Клиент **не выбирает** сервер вручную — все серверы включены в единую подписку автоматически.

## Д

- **Дашборд (Dashboard)** — См. Админка.

- **Устройство (Device / `VpnClient`)** — VPN-подключение клиента. В текущей версии клиент не создаёт устройства вручную через выбор страны/режима — устройства провизионятся автоматически при первом обращении Happ-клиента к subscription feed или через server-side bind к слотам `1..N`.

- **Доп. слот устройства (`DeviceSlotEntitlement`)** — Временное право на +1 устройство до конца оплаченного периода. Покупается отдельно или входит в расширенный тариф.

## Е

- **Единая подписка (Unified Subscription)** — Основной механизм доступа к VPN. Один токен на пользователя → все серверы (DE, DK, EE) включены в subscription feed. Happ-клиент автоматически получает список всех серверов и выбирает оптимальный. Клиент **не выбирает** страну, протокол или режим — маршрутизацией управляет система.

- **Единая ссылка** — URL `https://client.amonoraconnect.com/<token>`, который пользователь получает в боте. Открывается в браузере (HTML-страница со статусом, QR, install-flow) или в Happ (deep link для автоматического добавления подписки).

## К

- **Ключ (Config / Link)** — Строка подключения (начинается с `vless://` или `trojan://`). Настраивает VPN на устройстве. Генерируется автоматически для каждого сервера в единой подписке. Пользователь может скопировать individual connection URI для ручного импорта.

- **Конфиг** — То же, что и ключ. Файл или ссылка с настройками подключения: сервер, порт, протокол, UUID, транспорт.

## М

- **Миграция** — Скрипт обновления структуры базы данных. С апреля 2026: используется registry применённых шагов в PostgreSQL, а не слепой повторный прогон `ALTER`. Управляется через `backend/core/schema.py`.

- **Мост (Bridge)** — См. Bridge-user. Промежуточный слой для доступа без Telegram.

## Н

- **Нода (Node / `ManagedServer`)** — Сервер в другой стране, раздающий VPN-трафик. В Amonora три ноды:
  - **Германия** (`ffconnect.amonoraconnect.com`, `213.108.20.34`) — 3x-ui (Docker), VLESS + Reality + TCP. Основная.
  - **Дания** (`dk.amonoraconnect.com`, `81.17.159.58`) — Xray core (standalone), VLESS + Reality + XHTTP. Modern anti-DPI.
  - **Эстония** (`est.amonoraconnect.com`, `185.88.37.71`) — x-ui / Xray, VLESS + Reality + TCP. Скрытый reserve.

## П

- **Платёж (`PaymentRecord`)** — Запись об оплате в PostgreSQL. Типы: Telegram Stars, Platega (СБП/крипто авто), ручная оплата. Ручной платёж не даёт доступ до подтверждения через `@amonora_control_bot` или dashboard.

- **Platega** — Платёжный провайдер ([platega.com](https://platega.com)). Поддерживает авто-оплату через СБП и криптовалюту. Вебхук: `POST /webhooks/platega/{secret}`. Включается флагами `ENABLE_PLATEGA_SBP_USER_FLOW`, `ENABLE_PLATEGA_CRYPTO_USER_FLOW`.

- **Плейсхолдер** — Заглушка в тексте. Место, куда позже подставится реальное значение. Пример: `Привет, {name}!`.

- **Промокод (`PromoCode`)** — Код на скидку, дни доступа или подарочную подписку. Создаётся через dashboard. Обрабатывается через общий платёжный слой (`backend/core/promo_codes.py`).

- **Протокол** — Способ шифрования и передачи VPN-трафика. В Amonora: **VLESS** (основной), **Trojan** (альтернативный, для reserve на XUI-серверах). Транспорт: Reality + TCP / XHTTP. Клиент **не выбирает** протокол вручную — он определяется автоматически на основе сервера и режима.

- **Прод (Production)** — Боевой режим: реальный сервер, реальные пользователи, настоящие деньги. Противоположность — тест (`@test_amonora_bot`, dev-окружение).

- **Публичная подписка (`PublicSubscriptionRoute`)** — Модель в БД: один маршрут на комбинацию `(user_id, country_code, slot_index)`. Содержит `client_uuid`, `email`, `protocol`, `client_data` (JSON с VLESS/Trojan ссылками). Создаётся автоматически при первом обращении к subscription feed или при синхронизации доступа.

## Р

- **Реферал** — Пользователь, пришедший по приглашению другого. Приглашающий получает бонус на баланс: 50₽ (стандарт) или 100₽ (если реферал оплатил 12 месяцев).

- **Ротация** — Плановая замена или обновление. Например, «ротация ключей» — перевыдача конфигов для безопасности. В Amonora: deep repair — автоматическая перевыдача ключей через dashboard или `@amonora_control_bot`.

## С

- **Слот** — Место для устройства в подписке. По умолчанию `1..N`, где N = эффективный лимит устройств. Server-side bind к слотам сохраняет модель / ОС / версию ОС клиента.

- **Синтетический пользователь** — См. Bridge-user.

- **Subscription feed** — Текстовый payload, который отдаётся Happ-клиенту при запросе `/sub/<token>`. Содержит список всех серверов с connection URI. Happ автоматически добавляет эти серверы в список подключений пользователя.

- **Subscription page** — HTML-страница `client.amonoraconnect.com/<token>`, которую видит пользователь при открытии единой ссылки в браузере. Показывает статус подписки, срок, QR-код, инструкции по установке Happ.

## Т

- **Тариф** — Ценовой план: цена, срок, код тарифа. Определяется в runtime-конфиге + env. Текущие тарифы (RUB): 1 мес — 149₽, 3 мес — 399₽, 6 мес — 749₽, 12 мес — 1390₽. Тарифы в Stars: 1 мес — 299, 3 мес — 799, 6 мес — 1499, 12 мес — 2799.

- **Тикет (`SupportTicket`)** — Заявка в поддержку. Обращение клиента + история сообщений. Хранится в PostgreSQL без автоудаления. Создаётся через `@amonora_support_bot`.

- **Токен** — Цифровой ключ доступа. В Amonora: (1) Telegram-токен бота (`BOT_TOKEN`), (2) токен единой подписки (`PublicSubscriptionLink.token`) — URL-safe строка ~24 символа, определяет пользователя на subscription page.

- **Токенизированная страница подписки** — `client.amonoraconnect.com/<token>`. URL содержит токен, по которому сервер определяет подписку клиента. Браузер получает HTML со статусом, install-flow, QR. Happ (приложение) получает subscription feed с того же URL. Исходники: `client_ui/` (Vite + React + Mantine).

- **Триал (Trial)** — Бесплатный пробный период (3 дня, `TRIAL_DAYS=3`). Выдаётся один раз, привязан к подписке на канал `@amonora_new`.

- **Триггер (`ControlTriggerRule`)** — DB-driven правило автотриггеров. Автоматические действия по условиям: отправка уведомлений, рассылки, изменение статусов. Доставки логируются в `ControlTriggerDeliveryLog`.

## У

- **Узел** — См. Нода.

## Ф

- **Фича (Feature)** — Новая функция продукта. Например: «Мы завезли фичу — теперь можно платить криптой через Platega».

- **Фронтенд (Frontend)** — Видимая часть: кнопки в боте, страницы сайта, админка. В Amonora: `dashboard/ui` (Next.js 16), `client_ui` (Vite + React + Mantine), `landing` (Jinja2).

## Х

- **Хэппер (Happ)** — VPN-клиент, рекомендуемый для подключения. Happ автоматически понимает subscription feed и добавляет все серверы из единой подписки. Deep link: `happ://add/<page_url>`. Доступен для Android, iOS, Windows, macOS, Linux, Apple TV, Android TV.

## Ц

- **Ценообразование** — Логика расчёта стоимости. В Amonora: зависит от срока (1/3/6/12 мес) и метода оплаты (RUB / Stars). Тарифы задаются в env: `TARIFF_1M_RUB`, `TARIFF_1M_STARS` и т.д.

## Э

- **Эхо-бот** — Разговорное название `@test_amonora_bot`. Тестовый бот-клон для разработчиков. Отдаёт 8 тестовых конфигов только разрешённым Telegram ID. Работает независимо от основного бота, имеет свой `test_bot/router.py`.
