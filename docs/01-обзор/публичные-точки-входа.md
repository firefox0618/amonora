# 🌐 Публичные точки входа

Этот документ содержит список всех ресурсов, через которые пользователи и персонал взаимодействуют с системой Amonora. Используйте его как справочник ссылок и назначений.

## 1. Telegram-боты 🤖

Это основные интерфейсы взаимодействия. Все боты работают на одной платформе, но выполняют разные роли.

|Бот|Юзернейм / Ссылка|Назначение|Кто использует|
|---|---|---|---|
|**Клиентский бот**|[`@amonora_bot`](https://t.me/amonora_bot)|**Главный вход для клиентов.**  <br>• Покупка подписки  <br>• Получение ключей доступа  <br>• Выбор стран  <br>• Создание заявок в поддержку|Все клиенты|
|**Бот поддержки**|[`@amonora_support_bot`](https://t.me/amonora_support_bot)|**Канал связи с оператором.**  <br>• Прием обращений из клиентского бота  <br>• Чат с пользователем  <br>• Уведомление о статусе тикета|Клиенты + Операторы|
|**Контрольный бот**|[`@amonora_control_bot`](https://t.me/amonora_control_bot)|**Пульт управления админа.**  <br>• Алерты об ошибках системы  <br>• Уведомления об оплатах  <br>• Команды ручной остановки/запуска|Только Администраторы|
|**Тестовый бот**|[`@test_amonora_bot`](https://t.me/test_amonora_bot)|**Песочница для разработчиков.**  <br>• Проверка новых функций перед релизом  <br>• Отладка профилей подключения|Разработчики, Админы|

> ⚠️ **Важно:** Ссылки на `@amonora_control_bot` и `@test_amonora_bot` не публикуются в открытом доступе. Доступ к ним ограничен по ID пользователей.

## 3. Технические эндпоинты (Internal) 🔧

Ресурсы, недоступные из обычного интернета, но критически важные для работы системы. Доступны только с серверов проекта или через VPN администратора.

|Сервис|Назначение|Примечание|
|---|---|---|
|**Backend API**|Ядро системы. Обрабатывает запросы от ботов и панели.|FastAPI приложение|

|**VPN Панели (3x-ui)**|Локальные интерфейсы управления на самих нодах.|Порты закрыты фаерволом|

## 4. Сценарии перехода (User Flow)

Как пользователь перемещается между точками входа:

### Сценарий А: Новый клиент

1. Видит рекламу → Переходит на **Лендинг**.
2. Жмет кнопку «Попробовать» → Открывается **Клиентский бот**.
3. Жмет `/start` → Получает меню.
4. Выбирает «Подключить» → Бот выдает ключ.
5. Копирует ключ → Вставляет в приложение (V2Ray/Streisand).

### Сценарий Б: Проблема с подключением

1. Клиент в **Клиентском боте** жмет «Помощь».
2. Бот пишет: «Перейдите в бот поддержки» → Ссылка на **Бот поддержки**.
3. Клиент пишет проблему → Создается тикет.
4. Оператор видит тикет в **Боте поддержки** → Отвечает.

### Сценарий В: Авария на сервере

1. Мониторинг детектирует падение ноды.
2. **Контрольный бот** присылает алерт Админу: «⚠️ Нода DE-1 offline».
3. Админ кликает на кнопку в сообщении → Открывается **Панель управления**.
4. Админ перезапускает сервис или переключает трафик.

## 5. Безопасность точек входа

- **Публичные ресурсы** (Лендинг, Клиентский бот) защищены стандартными механизмами Telegram и HTTPS.
- **Приватные ресурсы** (Админка, Grafana, API) доступны только из внутренней сети или через SSH-туннель.
- **Доступ к ботам** регулируется на уровне кода: проверяется `user_id`. Если ID нет в списке админов — бот игнорирует команды управления.

> 🛡️ **Правило:** Никогда не передавайте ссылки на панель управления (`dashboard`) и контрольного бота посторонним лицам.
