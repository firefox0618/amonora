# 🔗 Публичные точки входа Amonora

Список всех публичных ресурсов, через которые пользователи взаимодействуют с сервисом.

---

## 🤖 Telegram-боты

### @amonora_bot — Клиентский бот (основной)
**Ссылка:** https://t.me/amonora_bot  
**Аудитория:** Все пользователи  
**Назначение:** Главный интерфейс для клиентов

**Что умеет:**
- ✅ Регистрация и получение пробного периода (3 дня)
- ✅ Выбор страны (Германия, Дания) и протокола (VLESS)
- ✅ Подключение устройств (до 1 на базовом тарифе)
- ✅ Оплата подписок (Stars, SBP, Crypto)
- ✅ Управление балансом и рефералами
- ✅ Получение конфигураций (QR, ссылки)
- ✅ Переход в поддержку при проблемах

**Команды:**
- `/start` — начать работу с ботом
- `/menu` — главное меню
- `/support` — перейти в поддержку

---

### @amonora_support_bot — Бот поддержки
**Ссылка:** https://t.me/amonora_support_bot  
**Аудитория:** Пользователи + поддержка  
**Назначение:** Обработка обращений в поддержку

**Что умеет:**
- ✅ Создание тикетов пользователями
- ✅ Переписка с админами поддержки
- ✅ Отправка файлов, скриншотов
- ✅ Уведомления о статусе тикета
- ✅ Закрытие тикетов

**Для пользователей:**
1. Нажмите `/start`
2. Опишите проблему
3. Прикрепите скриншоты (если нужно)
4. Ожидайте ответа поддержки

**Для поддержки:**
- Просмотр очереди тикетов в Dashboard
- Назначение ответственного
- Ведение переписки
- Закрытие тикета

---

### @amonora_control_bot — Контрол-бот
**Ссылка:** https://t.me/amonora_control_bot  
**Аудитория:** Только администраторы  
**Назначение:** Системные уведомления и алерты

**Что отправляет:**
- ⚠️ **Алерты** — критические события (падение нод, ошибки платежей)
- 💰 **Платежи на проверку** — ручные платежи (SBP, Crypto)
- 📊 **Дайджесты** — ежедневные/еженедельные отчёты
- 🔧 **Инциденты** — проблемы с доступом, ремонтом
- 📈 **Метрики** — важные изменения в показателях

**Категории уведомлений:**
| Категория | Пример | Срочность |
|-----------|--------|-----------|
| `access` | Пользователь не может подключиться | Средняя |
| `node` | Ноде DE недоступна | Высокая |
| `payment` | Ошибка обработки платежа | Средняя |
| `service` | Бот упал, база недоступна | Критическая |

### Страница подписки (Public Subscription Page)
**URL:** `https://ffconnect.amonoraconnect.com/<token>`  
**Аудитория:** Пользователи с активной подпиской  
**Назначение:** Просмотр и управление подпиской вне Telegram

**Возможности:**
- Просмотр статуса подписки
- QR-код для подключения
- Ссылки vless:// для импорта
- Информация об устройстве
- Срок действия доступа

**Как получить ссылку:**
1. В @amonora_bot выберите «Устройства»
2. Нажмите «Поделиться подпиской»
3. Скопируйте ссылку или QR-код

**Токен доступа:** Генерируется автоматически, хранится в таблице `public_subscription_links`

## 🎯 Пользовательские сценарии

### Сценарий 1: Новый пользователь
1. Видит рекламу → переходит на сайт
2. Сайт → кнопка «Запустить бота» → @amonora_bot
3. `/start` → регистрация
4. Подписка на канал → получение trial (3 дня)
5. Выбор страны (DE/DK) → получение конфига
6. Импорт в приложение → подключение

### Сценарий 2: Оплата подписки
1. Истёк trial → уведомление в боте
2. Выбор тарифа (1M/3M/6M/12M)
3. Выбор метода оплаты (Stars/SBP/Crypto)
4. Оплата → активация подписки
5. Продление доступа

### Сценарий 3: Проблема с подключением
1. Не работает VPN → бот не помогает
2. Кнопка «Поддержка» → @amonora_support_bot
3. Описание проблемы → создание тикета
4. Поддержка отвечает → решение
5. Тикет закрыт → пользователь доволен

### Сценарий 4: Админ получает алерт
1. Ноде DE упала → мониторинг фиксирует
2. Grafana → алерт в контрол-бот
3. @amonora_control_bot → уведомление админу
4. Админ исправляет → алерт снят
