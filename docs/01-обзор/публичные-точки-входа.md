# Публичные точки входа

Этот документ содержит полный список всех ресурсов, через которые пользователи, персонал и автоматизированные системы взаимодействуют с Amonora.

## 1. Telegram-боты

| Бот | Юзернейм / Ссылка | Назначение | Кто использует | Доступ |
|-----|-------------------|------------|----------------|--------|
| **Клиентский бот** | [`@amonora_bot`](https://t.me/amonora_bot) | Главный вход для клиентов: trial, единая подписка, оплата (Stars + Platega), рефералы, баланс, промокоды. V2-интерфейс: главное меню с 6 кнопками, система экранов. | Все клиенты | Публичный |
| **Бот поддержки** | [`@amonora_support_bot`](https://t.me/amonora_support_bot) | Приём обращений, создание тикетов, диалоги с клиентами, медиа (фото/видео/аудио), назначение ответственного, закрытие заявок | Клиенты + Операторы поддержки | Публичный (клиенты), приватный (операторы) |
| **Контрольный бот** | [`@amonora_control_bot`](https://t.me/amonora_control_bot) | Системные уведомления, review платежей (confirm/reject), 5-минутные коды входа в админку, рассылки, триггеры, node alerts | Только администраторы | Приватный (admin-only по Telegram ID) |
| **Тестовый бот** | [`@test_amonora_bot`](https://t.me/test_amonora_bot) | Проверка мобильных профилей: отдаёт 8 тестовых конфигов (Germany, Denmark, Estonia) по allowlist. Исходный код v2-интерфейса перенесён в основной бот. | Разработчики, админы | Приватный (admin-only по Telegram ID) |

> **Важно:** `@amonora_control_bot` и `@test_amonora_bot` не публикуются в открытом доступе. Доступ к ним ограничен через `ADMIN_IDS` / `SUPPORT_ADMIN_IDS` в `.env`.

## 3. Панель управления (Dashboard)

| Компонент | URL / Хост | Технологии | Доступ |
|-----------|-----------|------------|--------|
| **Dashboard backend** | `127.0.0.1:8088` (внутренний) | FastAPI, Python | Приватный (сервер-only, nginx) |
| **Dashboard frontend (UI)** | `127.0.0.1:3001` (внутренний) | Next.js 16, TypeScript | Приватный (сервер-only, nginx) |

Вход в админку: логин/пароль + 5-минутный Telegram-код через `@amonora_control_bot`.

Роли администраторов:
- **Владелец** — полный доступ
- **Тех. администратор** — управление серверами, пользователями, устройствами
- **Менеджер** — просмотр пользователей, платежи, поддержка

| **Дания** | `81.17.159.58` | `dk.amonoraconnect.com` | Xray core (standalone) | VLESS + Reality + XHTTP | Modern anti-DPI |
| **Эстония** | `185.88.37.71` | `est.amonoraconnect.com` | x-ui / Xray | VLESS + Reality + TCP | Reserve-регион |

VPN-ноды работают на отдельных серверах. На них **не** живут: боты, dashboard, PostgreSQL.

> **Клиент не выбирает сервер вручную.** Все три ноды автоматически включены в единую подписку. Happ-клиент получает список всех серверов и выбирает оптимальный.

## 8. Сценарии перехода (User Flow)

### Сценарий А: Новый клиент
1. Видит рекламу или рекомендацию → Переходит на **amonoraconnect.com**
2. Нажимает «Попробовать» → Открывается **@amonora_bot**
3. Нажимает `/start` → Бот показывает соглашение → пользователь принимает
4. Бот показывает пробный период → пользователь подписывается на канал `@amonora_new` → нажимает «Проверить»
5. Триал активирован (3 дня) → бот показывает главное меню
6. Нажимает **«Ключ»** → получает единую ссылку: `client.amonoraconnect.com/<token>`
7. Открывает ссылку → Happ автоматически добавляет подписку со всеми серверами (DE, DK, EE)

### Сценарий Б: Проблема с подключением
1. Клиент в **@amonora_bot** нажимает «Поддержка»
2. Бот пишет: «Перейдите в бот поддержки» → Ссылка на **@amonora_support_bot**
3. Клиент описывает проблему → Создаётся тикет с уникальным номером
4. Оператор видит тикет в **@amonora_support_bot** → Отвечает, ведёт диалог, при необходимости закрывает

### Сценарий В: Авария на сервере
1. Система детектирует проблему на ноде (например, Дания offline)
2. **@amonora_control_bot** присылает алерт админу: «⚠️ Нода DK offline»
3. Админ открывает **dashboard** → Проверяет статус сервера → Перезапускает сервис или переключает трафик
4. Админ подтверждает восстановление → Control-бот отправляет уведомление команде

### Сценарий Г: Покупка подписки
1. Клиент нажимает **«Продлить»** → видит список тарифов (1м / 3м / 6м / 12м)
2. Выбирает тариф → выбирает способ оплаты (СБП авто / СБП ручная / Крипто / Баланс / Stars)
3. При авто-оплате → переходит на checkout URL → оплачивает → webhook подтверждает → подписка активируется
4. При ручной оплате → видит реквизиты → оплачивает → нажимает «Оплатил» → заявка уходит на review
5. Админ подтверждает в **@amonora_control_bot** → подписка активируется
6. Единая ссылка обновляется → все серверы доступны
