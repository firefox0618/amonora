## 🔧 Компоненты системы

### 1. Telegram-боты

| Бот | Технология | Назначение |
|-----|------------|------------|
| **@amonora_bot** | aiogram 3.x | Основной клиентский бот |
| **@amonora_support_bot** | aiogram 3.x | Бот поддержки |
| **@amonora_control_bot** | aiogram 3.x | Уведомления для админов |
| **@test_amonora_bot** | aiogram 3.x | Тестовый бот (admin-only) |

**Файлы:**
- `bot/main.py` — точка входа клиентского бота
- `support_bot/` — бот поддержки
- `control_bot/` — контрол-бот
- `test_bot/` — тестовый бот

---

### 2. Backend Core

**Технологии:** Python 3.12+, FastAPI, SQLAlchemy (async)

**Основные модули:**
- `backend/core/models.py` — ORM-модели (40+ таблиц)
- `backend/core/schema.py` — миграции БД
- `backend/core/analytics.py` — аналитический движок
- `backend/core/promo_codes.py` — логика промокодов

**Функции:**
- Управление пользователями и подписками
- Проведение платежей
- Выдача VPN-конфигураций
- Сбор аналитики

---

### 3. Dashboard (Веб-панель)

**Технологии:** 
- Backend: FastAPI
- Frontend: Next.js 15, React, Tailwind CSS

**Компоненты:**
- `dashboard/main.py` — бэкенд админки
- `dashboard/ui/` — фронтенд на Next.js
- `dashboard/services.py` — бизнес-логика
- `dashboard/analytics.py` — отчёты и метрики

**Возможности:**
- Управление пользователями
- Обработка тикетов поддержки
- Финансы и аналитика
- Мониторинг нод
- Настройка тарифов

- **Тип:** Xray Core
- **Протокол:** VLESS
- **SSH:** `81.17.159.58:22`
- **Хост:** `dk.amonoraconnect.com`

## 🔄 Потоки данных

### Поток 1: Регистрация пользователя

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant B as @amonora_bot
    participant BE as Backend
    participant DB as PostgreSQL
    participant TG as Telegram
    
    U->>B: /start
    B->>BE: Проверка пользователя
    BE->>DB: SELECT user
    alt Новый пользователь
        BE->>DB: INSERT user
        BE->>TG: Проверка подписки на канал
        U->>TG: Подписаться
        TG->>BE: Подтверждение
        BE->>DB: Trial активирован
        BE->>B: Отправить welcome
    else Существующий
        BE->>B: Показать меню
    end
```

### Поток 2: Оплата подписки

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant B as @amonora_bot
    participant BE as Backend
    participant PG as Platega/CryptoPay
    participant DB as PostgreSQL
    
    U->>B: Выбрать тариф
    B->>BE: Создать счёт
    BE->>DB: payment_record (pending)
    BE->>PG: Создать инвойс
    PG->>U: Оплата
    U->>PG: Оплатить
    PG->>BE: Webhook (paid)
    BE->>DB: payment_record (paid)
    BE->>DB: subscription_expires_at += period
    BE->>B: Уведомить об оплате
    B->>U: Подписка активирована
```

### Поток 3: Выдача VPN-ключа

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant B as @amonora_bot
    participant BE as Backend
    participant DE as Node DE (3x-ui)
    participant DK as Node DK (Xray)
    participant DB as PostgreSQL
    
    U->>B: Выбрать страну
    B->>BE: Запрос конфигурации
    BE->>DB: Проверка подписки
    alt Страна = DE
        BE->>DE: Создать VLESS клиента
        DE->>BE: UUID + ссылка
    else Страна = DK
        BE->>DK: Создать VLESS клиента (SSH)
        DK->>BE: UUID + ссылка
    end
    BE->>DB: INSERT vpn_client
    BE->>B: QR + ссылка
    B->>U: Конфиг готов
```

## 🔐 Безопасность

### Границы доверия

| Зона | Доступ |
|------|--------|
| **Публичная** | Сайт, клиентский бот, страница подписки |
| **Пользовательская** | Личный кабинет, тикеты поддержки |
| **Админская** | Dashboard, контрол-бот, Grafana |
| **Инфраструктурная** | VPN-ноды, база данных, SSH |

### Защита данных

- Пароли хешируются (bcrypt)
- Сессионные куки с HttpOnly + Secure
- Webhook secret для проверки платежей
- Rate limiting на критичных эндпоинтах
- Изоляция нод через firewall
