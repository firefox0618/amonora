# Поток данных: Синхронизация устройств

Как устройства синхронизируются между backend, VPN-нодами и пользователем.

## 1. Sequence diagram: создание устройства

```mermaid
sequenceDiagram
    participant U as Пользователь
    participant Bot as @amonora_bot
    participant Hdl as bot/handlers/devices.py
    participant DB_FN as bot/db.py
    participant Prov as bot/vpn_provisioning.py
    participant XUI as bot/vpn_api.py (XUIClient)
    participant V3x as 3x-ui (Германия/Эстония)
    participant Xray as Xray core (Дания, SSH)
    participant PG as PostgreSQL
    participant Comp as bot/device_compensation.py
    participant Analytics as backend.core.analytics

    U->>Bot: Нажимает "Добавить устройство"
    Bot->>Hdl: device selection flow

    Note over Hdl: FSM state machine: OS → Country → Protocol → Name

    Hdl->>Hdl: ask_device_os() → пользователь выбирает OS
    Hdl->>Hdl: ask_device_country() → пользователь выбирает страну
    Hdl->>Hdl: ask_device_protocol() → VLESS/Trojan
    Hdl->>Hdl: ask_device_name() → пользователь вводит имя

    Hdl->>Bot: _create_device_from_selection(callback, state, country_code, mode)

    Hdl->>DB_FN: get_user_by_telegram_id(telegram_id)
    DB_FN->>PG: SELECT users WHERE telegram_id = ?
    PG-->>DB_FN: user

    Hdl->>Hdl: Проверка: mode_available_for_user(mode)
    Hdl->>Hdl: Проверка: get_access_expires_at(user_id) — есть ли доступ
    Hdl->>Hdl: Проверка: region capacity — не переполнена ли нода
    Hdl->>Hdl: Проверка: mode_supported_in_region(mode, country_code)

    Hdl->>Analytics: safe_emit_analytics_event(EVENT_CONFIG_REQUESTED)

    alt protocol = "vless"
        Hdl->>Prov: get_vless_provisioner(country_code, provider_type)
        Prov->>V3x: POST /login (3x-ui API auth)
        V3x-->>Prov: session cookie
        Prov->>V3x: POST /panel/api/inbounds/addClient
        V3x-->>Prov: client created (id, uuid, email)
    else protocol = "trojan"
        Hdl->>XUI: XUIClient(country_code)
        XUI->>V3x: POST /login
        V3x-->>XUI: session cookie
        XUI->>V3x: find_inbound("trojan", 8443)
        V3x-->>XUI: inbound info
        XUI->>V3x: provision_trojan_client(user_id, email, access_expires_at)
        V3x-->>XUI: client created
    end

    Hdl->>DB_FN: create_vpn_client(user_id, protocol, client_uuid, email, xui_client_id, client_data)
    DB_FN->>PG: INSERT INTO vpn_clients (protocol, client_uuid, email, xui_client_id, client_data)
    PG-->>DB_FN: new vpn_client row

    Hdl->>Analytics: safe_emit_analytics_event(EVENT_CONFIG_ISSUED)
    Hdl->>Hdl: generate_qr_image(config_link)
    Hdl->>U: QR-код + ссылка подключения + инструкция

    Note over Comp: enqueue_finalize_created_device_job()
    Hdl->>Comp: enqueue_finalize_created_device_job(vpn_client_id)
    Comp->>PG: INSERT INTO device_compensation_jobs (action="finalize_created_device", status="pending")
```

```mermaid

        Note over Hdl: Удалить на старой ноде, создать на новой
        Hdl->>V3xOld: delete_vless_client(old_client_uuid)
        V3xOld-->>Hdl: {success: true}

        Hdl->>V3xNew: addClient(new_country, email)

        Hdl->>DB_FN: update_vpn_client_metadata(device_id, {country_code, xui_client_id, client_data})
        DB_FN->>PG: UPDATE vpn_clients SET client_data = ? WHERE id = ?
    else same-region change
        Hdl->>Hdl: Обновить только metadata
        Hdl->>DB_FN: update_vpn_client_metadata(device_id, {country_code})
    end

    Hdl->>U: "Страна изменена на {country_name}" + новая ссылка
```

## 4. Sequence diagram: device compensation (асинхронные задачи)

```mermaid
sequenceDiagram
    participant Sys as bot (фоновый процесс)
    participant Comp as bot/device_compensation.py
    participant PG as PostgreSQL
    participant Prov as bot/vpn_provisioning.py
    participant V3x as 3x-ui

    Note over Sys: Периодическая обработка compensation jobs

    Sys->>Comp: process_pending_compensation_jobs()
    Comp->>PG: SELECT device_compensation_jobs WHERE status = "pending" AND next_attempt_at <= now()
    PG-->>Comp: list of jobs

    loop для каждого job
        Comp->>Comp: lock job (locked_at = now())
        Comp->>Comp: attempt_count += 1

        alt action = "finalize_created_device"
            Comp->>PG: SELECT vpn_clients WHERE id = job.vpn_client_id
            PG-->>Comp: vpn_client
            Comp->>Comp: Проверить что клиент существует на ноде
        else action = "cleanup_created_device"
            Comp->>Prov: delete_vless_client(client_uuid, email)
            Prov->>V3x: DELETE client
            V3x-->>Prov: OK/ERROR
        else action = "restore_deleted_device"
            Comp->>Prov: restore_vless_client(client_uuid, email, access_expires_at)
            Prov->>V3x: ADD client
            V3x-->>Prov: OK/ERROR
        end

        alt success
            Comp->>PG: UPDATE device_compensation_jobs SET status = "completed", completed_at = now()
        else failure
            alt attempt_count < MAX_ATTEMPTS (10)
                Comp->>PG: UPDATE device_compensation_jobs SET status = "pending", next_attempt_at = now() + delay, last_error = ?
            else max attempts reached
                Comp->>PG: UPDATE device_compensation_jobs SET status = "failed", last_error = ?
            end
        end
    end
```

## 5. Лимиты устройств

| Параметр | Значение | Конфиг |
|----------|----------|--------|
| Устройства по умолчанию | 1 | `VPN_MAX_DEVICES_PER_KEY` |
| Максимальный лимит | 5 + базовый | `DEVICE_SLOT_MAX_EXTRA_SLOTS` |
| Цена дополнительного слота | 49 руб/мес | `DEVICE_SLOT_UNIT_PRICE_RUB` |
| Анти-шаринг lease | 180 сек | `VPN_ANTISHARING_LEASE_SECONDS` |
| Anti-sharing soft limit | Включён | `VPN_ANTISHARING_SOFT_LIMIT_ENABLED` |

### Как работает лимит

1. При создании устройства: `count_user_vpn_clients(user_id)` → проверка лимита
2. `get_device_limit_for_user(user_id)` = базовый лимит + активные `device_slot_entitlements`
3. Если лимит достигнут — предлагается купить дополнительные слоты
4. Каждый купленный слот: `device_slot_entitlements` запись с `expires_at`

## 6. Регионы и ноды

| Страна | Код | Хост | Runtime | Протоколы |
|--------|-----|------|---------|-----------|
| Германия | `de` | ffconnect.amonoraconnect.com | 3x-ui | VLESS, Trojan |
| Дания | `dk` | dk.amonoraconnect.com | Xray core (standalone) | VLESS (modern anti-DPI) |
| Эстония | `ee` | est.amonoraconnect.com | x-ui / Xray | VLESS (резерв) |

### Управление нодами

- **Германия/Эстония:** HTTP API к 3x-ui (`XUIClient`)
- **Дания:** SSH подключение → редактирование config.json + meta → перезапуск Xray
  - Конфиг: `XRAY_CORE_DK_CONFIG_PATH` = `/usr/local/etc/xray/config.json`
  - Мета: `XRAY_CORE_DK_META_PATH` = `/usr/local/etc/xray/amonora_dk_meta.json`
  - IP enforcement: `ops/xray_single_ip_enforcer.py`

## 7. Режимы подключения

| Режим | Описание | Протокол |
|-------|----------|----------|
| `stable` | Стандартный режим | VLESS |
| `anti-dpi` | Обход DPI (Дания) | VLESS (Xray core) |
| `mobile` | Для мобильных приложений | VLESS/Trojan |

**Файлы:** `bot/utils/modes.py`
- `resolve_effective_mode(mode, country_code)` — финальный режим
- `mode_available_for_user(mode, telegram_id)` — проверка доступа
- `mode_supported_in_region(mode, country_code)` — поддержка в регионе

## 8. Используемые таблицы БД

| Таблица | Операция |
|---------|----------|
| `vpn_clients` | INSERT при создании, DELETE при удалении, UPDATE metadata |
| `vpn_client_activations` | INSERT при активации на устройстве (fingerprint) |
| `device_slot_entitlements` | SELECT при проверке лимита |
| `device_compensation_jobs` | INSERT/UPDATE при асинхронной компенсации |
| `users` | SELECT для проверки access_expires_at, subscription_status |
| `analytics_events` | INSERT: `config_requested`, `config_issued`, `config_issue_failed`, `connection_ready`, `connection_failed` |

## 9. Ограничения

| Ограничение | Описание |
|-------------|----------|
| **Лимит устройств** | По умолчанию 1 устройство на пользователя, расширяется покупкой слотов |
| **Anti-sharing** | Lease-based detection (180 сек) — если один конфиг используется на разных IP, возможна блокировка |
| **Емкость нод** | При переполнении ноды — отказ в создании устройства |
| **Режимы по регионам** | Не все режимы доступны во всех регионах (mobile mode — ограниченный rollout) |
| **Cross-region change** | Смена страны = удаление на старой ноде + создание на новой (не мгновенно) |
| **Fingerprint** | `vpn_client_activations` отслеживает fingerprint_hash для дедупликации устройств |
| **Access expiry** | VPN-клиент создаётся с `access_expires_at` — нода может отключить по истечении |
