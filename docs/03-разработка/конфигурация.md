# Конфигурация Amonora

Все переменные окружения загружаются через `bot/config.py` с помощью `dotenv`.

> **Правило:** Обязательные переменные вызывают `ValueError` при отсутствии. Опциональные имеют значения по умолчанию.

---

## Bot Tokens

| Переменная | Где используется | Описание | Пример |
|------------|-----------------|----------|--------|
| `BOT_TOKEN` | `bot/config.py`, `bot/main.py` | Токен основного Telegram-бота (@amonora_bot). Обязательная. | `123456:ABC-DEF...` |
| `ADMIN_IDS` | `bot/config.py` | Список Telegram ID администраторов (comma-separated). Обязательная. | `123456789,987654321` |
| `SUPPORT_BOT_TOKEN` | `bot/config.py`, `support_bot/main.py` | Токен бота поддержки. Опциональная (если пустая — support_bot не запустится). | `654321:ABC-DEF...` |
| `SUPPORT_ADMIN_IDS` | `bot/config.py` | Список Telegram ID админов поддержки. По умолчанию = `ADMIN_IDS`. | `123456789` |
| `AMONORA_TEST_BOT_TOKEN` | `bot/config.py`, `test_bot/main.py` | Токен тестового бота (v2/develop). Опциональная. | `111222:ABC-DEF...` |

| `AMONORA_CONTROL_BOT_TOKEN` | `bot/config.py`, `control_bot/main.py` | Токен контрольного бота (административный). Опциональная. | `333444:ABC-DEF...` |
| `AMONORA_CONTROL_OWNER_IDS` | `bot/config.py` | ID владельцев контрольного бота. | `123456789` |
| `AMONORA_CONTROL_ADMIN_IDS` | `bot/config.py` | ID администраторов контрольного бота. | `123456789` |
| `AMONORA_CONTROL_OPERATOR_IDS` | `bot/config.py` | ID операторов контрольного бота. | `123456789` |
| `AMONORA_CONTROL_SUPPORT_VIEW_ONLY_IDS` | `bot/config.py` | ID поддержки (только просмотр) в контрольном боте. | `123456789` |
| `AMONORA_CONTROL_ALLOWED_TELEGRAM_IDS` | `bot/config.py` | Все допущенные ID контрольного бота. По умолчанию = owner + admin + operator + support. | `123456789` |

## Database

| Переменная | Где используется | Описание | Пример |
|------------|-----------------|----------|--------|
| `DB_HOST` | `bot/config.py`, `backend/core/database.py` | Хост PostgreSQL. Обязательная. | `localhost` |
| `DB_PORT` | `bot/config.py` | Порт PostgreSQL. Обязательная. | `5432` |
| `DB_NAME` | `bot/config.py` | Имя базы данных. Обязательная. | `amonora_db` |
| `DB_USER` | `bot/config.py` | Пользователь PostgreSQL. Обязательная. | `amonora` |
| `DB_PASS` | `bot/config.py` | Пароль PostgreSQL. Обязательная. | `secure_password` |
| `AUTO_APPLY_SCHEMA` | `backend/core/schema.py` | Автоматическое применение схемы при старте. По умолчанию = `1` (включено). | `1` |

URL базы данных собирается автоматически в `config.database_url`:
```
postgresql+asyncpg://{DB_USER}:{DB_PASS}@{DB_HOST}:{DB_PORT}/{DB_NAME}
```

| Переменная | Где используется | Описание | Пример |
|------------|-----------------|----------|--------|
| `AMONORA_CONTROL_ENABLE_PAYMENTS` | `bot/config.py` | Включить секцию платежей в control_bot. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_ENABLE_NODES` | `bot/config.py` | Включить секцию нод. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_ENABLE_USERS` | `bot/config.py` | Включить секцию пользователей. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_ENABLE_ACCESS` | `bot/config.py` | Включить секцию доступа. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_ENABLE_PANEL_AUTH` | `bot/config.py` | Включить секцию аутентификации панели. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_ENABLE_ERRORS` | `bot/config.py` | Включить секцию ошибок. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_ENABLE_SYSTEM` | `bot/config.py` | Включить секцию системы. По умолчанию = `1`. | `1` |
| `AMONORA_CONTROL_DEFAULT_COOLDOWN_SECONDS` | `bot/config.py` | Стандартный кулдаун команд (сек). По умолчанию = `300`. | `300` |
| `AMONORA_CONTROL_INFRA_COOLDOWN_SECONDS` | `bot/config.py` | Кулдаун инфраструктурных команд. По умолчанию = `900`. | `900` |
| `AMONORA_CONTROL_NIGHT_CRITICAL_ONLY` | `bot/config.py` | Ночью только критические уведомления. По умолчанию = `0`. | `0` |
| `AMONORA_CONTROL_NIGHT_HOURS` | `bot/config.py` | Ночные часы (диапазон). По умолчанию = `00-07`. | `00-07` |
| `AMONORA_CONTROL_DAILY_SUMMARY_ENABLED` | `bot/config.py`, `control_bot/main.py` | Включить ежедневную сводку. По умолчанию = `0`. | `0` |
| `AMONORA_CONTROL_DAILY_SUMMARY_HOUR` | `bot/config.py`, `control_bot/main.py` | Час ежедневной сводки. По умолчанию = `9`. | `9` |

## Как работает загрузка конфигурации

`bot/config.py` использует следующую логику:

```python
# Обязательная переменная — вызывает ValueError если отсутствует
bot_token = get_env("BOT_TOKEN")

# Опциональная — возвращает default или None
openai_api_key = get_optional_env("OPENAI_API_KEY", "")

# Опциональная с фолбэком
trial_days = int(get_env("TRIAL_DAYS", "3"))

# Boolean-флаг — принимает "1", "true", "yes", "on"
enable_platega_sbp_user_flow = (
    get_optional_env("ENABLE_PLATEGA_SBP_USER_FLOW", "1").strip().lower()
    in {"1", "true", "yes", "on"}
)

# Список int через запятую
admin_ids = parse_int_list(get_env("ADMIN_IDS"))

# Список строк через запятую
accepted_assets = parse_str_list(get_env("CRYPTO_PAY_ACCEPTED_ASSETS"))
```
