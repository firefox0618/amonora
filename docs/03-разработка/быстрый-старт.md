# Быстрый старт разработки Amonora

## Предусловия

### Системные требования

| Компонент | Версия / Требование |
|-----------|---------------------|
| Python | 3.11+ (asyncio, dataclasses, zoneinfo) |
| PostgreSQL | 14+ (asyncpg драйвер) |
| Node.js | 18+ (для dashboard UI — Next.js) |
| ОС | Linux (Ubuntu/Debian рекомендуется) |

### Обязательные токены и доступы

Перед первым запуском необходимо получить:

- **BOT_TOKEN** — токен основного Telegram-бота (`@amonora_bot`)
- **SUPPORT_BOT_TOKEN** — токен бота поддержки (`@amonora_support_bot`)
- **AMONORA_CONTROL_BOT_TOKEN** — токен контрольного бота (для администраторов)
- **AMONORA_TEST_BOT_TOKEN** — токен тестового бота (для v2/develop)
- **ADMIN_IDS** — comma-separated список Telegram ID администраторов

### Внешние сервисы

- **PostgreSQL** — рабочая база данных
- **3x-ui панель** — VPN-панель управления (x-ui) для Германии (`DE`)
- **Xray Core** — VPN-нода для Дании (`DK`) через SSH
- **Platega** (опционально) — платёжный провайдер SBP/крипто
- **Crypto Pay** (опционально) — платёжный провайдер через Telegram Crypto Bot

## Шаг 4: Настройка .env

Скопируй шаблон и заполни переменные:

```bash
cp .env.example .env
```

Минимальный набор для запуска бота:

```env
# Telegram bots
BOT_TOKEN=your_main_bot_token
ADMIN_IDS=123456789

# Database
DB_HOST=localhost
DB_PORT=5432
DB_NAME=amonora_db
DB_USER=amonora
DB_PASS=your_db_password

# XUI / VPN panel
XUI_URL=http://127.0.0.1:12053
XUI_USERNAME=admin
XUI_PASSWORD=admin_password

# Product defaults
CHANNEL_ID=-1001234567890
TRIAL_DAYS=3
VPN_HOST=ffconnect.amonoraconnect.com
```

Полный список переменных — в [конфигурация.md](конфигурация.md).

## Шаг 5: Запуск компонентов

### 5.1 Основной бот

```bash
source venv/bin/activate
python -m bot.main
```

Команды бота после запуска:
- `/start` — Открыть Amonora
- `/menu` — Главное меню
- `/support` — Поддержка

Бот регистрирует 9 роутеров (в порядке подключения):
1. `test_bot.router` — v2 роутер
2. `start` — команда /start, триал
3. `cabinet` — личный кабинет
4. `devices` — управление устройствами
5. `protocol` — выбор протокола
6. `info` — информация
7. `referrals` — реферальная программа
8. `support` — обращения в поддержку
9. `tariffs` — просмотр и покупка тарифов

### 5.2 Dashboard (API + UI)

```bash
source venv/bin/activate
uvicorn dashboard.main:app --host 0.0.0.0 --port 8088 --reload
```

Dashboard доступен по адресу: `http://127.0.0.1:8088`

Логин через двухфакторную аутентификацию:
1. Ввести username/password в `/login`
2. Получить код в Telegram (отправляется через control_bot)
3. Ввести код в `/verify`

Роли администраторов: `owner`, `tech`, `support` (настраиваются через ENV).

### 5.3 Landing (публичная страница)

```bash
source venv/bin/activate
uvicorn landing.main:app --host 0.0.0.0 --port 8000 --reload
```

Landing доступен по адресу: `http://127.0.0.1:8000`

Публичные эндпоинты:
- `GET /` — главная страница с тарифами
- `GET /manual` — инструкция для пользователей
- `GET /legal/{page}` — юридические документы (contacts, privacy, terms, refunds, license, cookies)
- `GET /health` — healthcheck
- `POST /bridge/access` — выдача временного VPN-ключа на 24 часа
- `POST /webhook/platega` — вебхук Platega
- `POST /webhook/crypto-pay` — вебхук Crypto Pay
- `GET /sub/{token}` — подписка (subscription feed)
- `GET /happ/add` — Happ wrapper для автоподключения

### 5.4 Support Bot

source venv/bin/activate

## Шаг 6: Production запуск (systemd)

В production все сервисы запускаются через systemd. Файлы сервисов лежат в `ops/systemd/`:

| Сервис | Файл | Описание |
|--------|------|----------|
| Основной бот | (отсутствует, запускается вручную или в рамках общего процесса) | |
| Dashboard UI | `amonora-dashboard-ui.service` | Next.js frontend, port 3001 |
| Control Bot | `amonora-control-bot.service` | Административный бот |
| Test Bot | `amonora-test-bot.service` | Тестовый бот |
| Access Reminders | `amonora-access-reminders.service` + `.timer` | Таймер напоминаний о доступе |
| Analytics Refresh | `amonora-analytics-refresh.service` + `.timer` | Таймер обновления аналитики |
| Server Watchdog | `amonora-server-watchdog.service` + `.timer` | Мониторинг серверов |
| Grafana | `amonora-grafana.service` | Grafana integration |
| Grafana DB Tunnel | `amonora-grafana-db-tunnel.service` | Туннель к БД Grafana |
| DK MTG | `amonora-dk-mtg.service` | Denmark MTG tunnel |
| DK IP Enforcer | `amonora-dk-single-ip-enforcer.service` + `.timer` | Denmark IP whitelist |
| PG Backup | `amonora-core-pg-backup.service` + `.timer` | Бэкап PostgreSQL |
| Remote Artifact Backup | `amonora-remote-artifact-backup@.service` + `.timer` | Бэкап артефактов |
| Restore Validation | `amonora-restore-validation.service` + `.timer` | Валидация бэкапов |
| XUI Tunnel | `amonora-xui-tunnel.service` | Туннель к 3x-ui |

Пример содержимого `amonora-control-bot.service`:

```ini
[Unit]
Description=Amonora Control Bot
After=network.target postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/amonora_bot
Environment=PYTHONPATH=/opt/amonora_bot
ExecStart=/opt/amonora_bot/venv/bin/python -m control_bot.main
Restart=always
RestartSec=5
TimeoutStopSec=20

[Install]
WantedBy=multi-user.target
```

Пример содержимого `amonora-dashboard-ui.service`:

```ini
[Unit]
Description=Amonora Панель управления UI (Next.js)
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/amonora_bot/dashboard/ui
EnvironmentFile=-/etc/amonora-dashboard-ui.env
ExecStart=/usr/local/bin/node ./node_modules/next/dist/bin/next start -H 127.0.0.1 -p 3001
Restart=always
RestartSec=5
TimeoutStopSec=20

[Install]
WantedBy=multi-user.target
```

Установка сервисов:

```bash
sudo cp ops/systemd/*.service /etc/systemd/system/
sudo cp ops/systemd/*.timer /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable amonora-control-bot
sudo systemctl start amonora-control-bot
```

## Troubleshooting первых шагов

### Ошибка: `Environment variable BOT_TOKEN is not set`

**Причина:** Файл `.env` не найден или пустой `BOT_TOKEN`.

**Решение:**
```bash
cat .env | grep BOT_TOKEN
# Если пусто — заполни .env
```

### Ошибка: `connection refused` к PostgreSQL

**Причина:** PostgreSQL не запущен или неверный `DB_HOST`.

**Решение:**
```bash
sudo systemctl status postgresql
sudo systemctl start postgresql
```

### Ошибка: `3x-ui login failed`

**Причина:** Неверные `XUI_URL`, `XUI_USERNAME` или `XUI_PASSWORD`.

**Решение:**
```bash
curl -v http://127.0.0.1:12053/login
# Проверь что панель доступна
```

### Ошибка: `asyncpg` или `psycopg2` не устанавливается

**Причина:** Отсутствуют системные зависимости для компиляции.

**Решение:**
```bash
sudo apt-get install -y libpq-dev python3-dev build-essential
pip install -r requirements.txt
```

### Dashboard не запускается — `Address already in use`

**Причина:** Port 8088 уже занят.

**Решение:**
```bash
lsof -i :8088
# Найди процесс и либо убей его, либо измени DASHBOARD_PORT в .env
```

### Бот не отправляет сообщения

**Причина:** Нет `CHANNEL_ID` или бот не добавлен в канал как администратор.

**Решение:**
1. Создай канал в Telegram
2. Добавь бота как администратора
3. Укажи `CHANNEL_ID=-100xxxxxxxxxx` в `.env`

### Schema не применяется автоматически

**Причина:** `AUTO_APPLY_SCHEMA=0` или нет прав на DDL.

**Решение:**
```bash
# Убедись что AUTO_APPLY_SCHEMA=1 в .env
# Или запусти вручную:
python -c "
from backend.core.schema import ensure_schema
import asyncio
asyncio.run(ensure_schema())
"
```
