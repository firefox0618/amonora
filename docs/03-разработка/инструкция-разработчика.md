# Инструкция разработчика

## Структура серверов

| Директория | Назначение |
|------------|-----------|
| `/srv/amonora_repo_clean` | Git-репозиторий (ветка `main`). Связан с `git@github.com:firefox0618/amonora.git` |
| `/opt/amonora_bot` | Runtime — отсюда запускаются все сервисы. Заполняется через `deploy.sh` |
| `/opt/amonora_backup` | Бекапы: `predeploy_*.tar.gz` (последние 3) |

## Скрипты

| Скрипт | Что делает |
|--------|-----------|
| `/opt/deploy.sh` | Обновляет репо из `origin/main`, синхронизирует код в runtime, перезапускает сервисы, проверяет здоровье |
| `/opt/backup.sh` | Создаёт тарбол runtime, хранит последние 3 бекапа |
| `/opt/rollback.sh` | Откат к предыдущей версии |

### Порядок работы

```bash
# 1. Локально — commit + push
git add -A
git commit -m "..."
git push origin main

# 2. На сервере — бекап + деплой
/opt/backup.sh && /opt/deploy.sh

# Или только деплой (если бекап уже сделан)
/opt/deploy.sh

# Или только бекап
/opt/backup.sh
```

## Сервисы (systemd)

| Сервис | Что делает |

| `amonora-bot.service` | Клиентский бот `@amonora_bot` |

| `amonora-control-bot.service` | Контрол-бот `@amonora_control_bot` |

## Архитектура бота

### bot/router.py — основной UI-роутер (~4850 строк)

Содержит **весь пользовательский интерфейс** клиентского бота:
- `/start`, `/menu` — команды
- Главное меню (6 кнопок): подписка, ключ, продлить, информация, поддержка, бонусы
- Onboarding: соглашение → триал → готовность
- Единая подписка: `client.amonoraconnect.com/<token>` + Happ
- Платежи: тариф → способ оплаты → подтверждение
- Рефералы, баланс, промокоды, подарочные подписки

### bot/main.py — точка входа

```python
dp.include_router(main_router)    # bot/router.py — ПЕРВЫЙ
dp.include_router(start_router)   # legacy (недостижим)
dp.include_router(devices_router) # legacy (недостижим)
# ... остальные legacy-роутеры
```

`main_router` подключён первым и перехватывает все пользовательские взаимодействия.

### bot/handlers/ — legacy

Старые обработчики (`start.py`, `devices.py`, `protocol.py`, `tariffs.py`) **недостижимы** для обычных пользователей. Сохраняются как shared-логика для dashboard и внутренних вызовов.

### bot/assets/v2/ — картинки экранов

Каждый экран бота использует картинку `sakura_<screen_key>.jpg`:
- `sakura_main_menu.jpg`, `sakura_agreement.jpg`, `sakura_trial.jpg`, `sakura_support.jpg`, `sakura_bonus.jpg`, `sakura_finish.jpg`, `sakura_instruction.jpg`, `sakura_my_subscription.jpg`, `sakura_emblem.jpg`

**Важно:** Эти картинки должны присутствовать в `bot/assets/v2/` как в репо, так и в runtime. `deploy.sh` автоматически копирует их.

## Единая подписка

Клиент получает **один токен** → все серверы (DE, DK, EE) включены автоматически. Клиент **не выбирает** страну, протокол или режим — маршрутизацией управляет система.

- Subscription page: `client.amonoraconnect.com/<token>`
- Happ feed: `client.amonoraconnect.com/sub/<token>`
- Happ wrapper: `client.amonoraconnect.com/happ/add?sub=...`

Подробнее: [02-архитектура/потоки-данных/unified-subscription-flow.md](../02-архитектура/потоки-данных/unified-subscription-flow.md)

## Чего НЕ делать

- **Не править** `/opt/amonora_bot` вручную (только через git + deploy)
- **Не удалять** `/opt/amonora_bot/.env` (там секреты)
- **Не трогать** `/opt/amonora_bot/venv/` (зависимости)
- **Не коммитить** `.env`, `.ssh/`, `backups/`, `venv/`
