# Как обновить тарифы

## Где определены тарифы

Тарифы определены в **двух местах**:

### 1. Переменные окружения (`.env`)

В `bot/config.py` тарифы загружаются из ENV:

```python
# RUB тарифы
tariff_1m_rub=int(get_env("TARIFF_1M_RUB", "149")),
tariff_3m_rub=int(get_env("TARIFF_3M_RUB", "399")),
tariff_6m_rub=int(get_env("TARIFF_6M_RUB", "749")),
tariff_12m_rub=int(get_env("TARIFF_12M_RUB", "1390")),

# Stars тарифы
tariff_1m_stars=int(get_env("TARIFF_1M_STARS", "299")),
tariff_3m_stars=int(get_env("TARIFF_3M_STARS", "799")),
tariff_6m_stars=int(get_env("TARIFF_6M_STARS", "1499")),
tariff_12m_stars=int(get_env("TARIFF_12M_STARS", "2799")),
```

### 2. Runtime-конфигурация (dashboard services)

В `dashboard/services.py` есть функция `get_runtime_tariffs()` и `get_runtime_tariffs_list()` которые возвращают тарифы из runtime-состояния (кэш).

Обновление через dashboard API:
```
POST /dashboard/api/v2/settings/tariffs
POST /dashboard/services/tariffs
```

### 3. Модели (тарифы как объект)

В `bot/utils/tariffs.py` (требуется уточнение точного пути) определены:
- `get_tariff(code)` — получить тариф по коду
- `get_tariffs_list()` — список всех тарифов

Тариф содержит:
- `code` — код тарифа (`1m`, `3m`, `6m`, `12m`)
- `title` — название
- `rub_price` — цена в рублях
- `stars_price` — цена в Stars
- `duration_days` — длительность в днях

Текущие значения (из `.env` по умолчанию):

| Код | Название | RUB | Stars | Дней |
|-----|----------|-----|-------|------|
| `1m` | 1 месяц | 149 | 299 | 30 |
| `3m` | 3 месяца | 399 | 799 | 90 |
| `6m` | 6 месяцев | 749 | 1499 | 180 |
| `12m` | 12 месяцев | 1390 | 2799 | 365 |

## Как изменить цены

### Способ 1: Через .env (рекомендуется для production)

1. Открой `.env`:
   ```bash
   nano /opt/amonora_bot/.env
   ```

2. Измени нужные переменные:
   ```env
   TARIFF_1M_RUB=199
   TARIFF_3M_RUB=499
   TARIFF_6M_RUB=899
   TARIFF_12M_RUB=1590
   ```

3. Перезапусти бота:
   ```bash
   # Если работает как systemd сервис — рестарт невозможен без unit-файла
   # Для dev-режима:
   # Останови бота (Ctrl+C)
   python -m bot.main
   ```

### Способ 2: Через Dashboard API

```bash
curl -X POST http://127.0.0.1:8088/dashboard/api/v2/settings/tariffs \
  -H "Content-Type: application/json" \
  -H "Cookie: amonora_dashboard_session=<your_session_token>" \
  -d '{
    "tariff_1m_rub": 199,
    "tariff_3m_rub": 499,
    "tariff_6m_rub": 899,
    "tariff_12m_rub": 1590
  }'
```

**Response:**
```json
{
  "ok": true,
  "notice": "Тарифы обновлены"
}
```

> **Примечание:** Через API v2 обновляется runtime-кэш, но НЕ `.env` файл. После перезапуска бота значения вернутся из `.env`.

### Способ 3: Через Dashboard UI

1. Зайди в Dashboard → Settings → Tariffs
2. Введи новые цены
3. Нажми "Сохранить"

## Нужно ли обновлять миграции

**Нет**, тарифы не хранятся в БД как схема. Это runtime-конфигурация.

Миграции (`backend/core/schema.py`) не затрагивают тарифы. Миграции касаются только структуры таблиц.

Однако, если ты добавляешь **новый тарифный код** (например, `18m`):
1. Добавь новые ENV-переменные: `TARIFF_18M_RUB`, `TARIFF_18M_STARS`
2. Обнови `bot/utils/tariffs.py` — добавь новый тариф в список
3. Обнови `bot/config.py` — добавь поля в Config dataclass

## Как проверить что цены обновились

### Через бота

1. Отправь `/menu` → `Тарифы`
2. Проверь что отображаются новые цены

### Через Dashboard

```bash
curl http://127.0.0.1:8088/dashboard/api/v2/settings \
  -H "Cookie: amonora_dashboard_session=<token>"
```

### Через Python

```python
python -c "
from bot.config import config
print(f'1m: {config.tariff_1m_rub} RUB')
print(f'3m: {config.tariff_3m_rub} RUB')
print(f'6m: {config.tariff_6m_rub} RUB')
print(f'12m: {config.tariff_12m_rub} RUB')
"
```

## Где кэшируются тарифы

### Runtime кэш dashboard

В `dashboard/services.py`:
- `get_runtime_tariffs()` — читает из runtime-состояния
- `update_tariffs()` — обновляет runtime-состояние

Кэш очищается при перезапуске dashboard.

### Кэш landing страницы

Landing (`landing/main.py`) загружает тарифы напрямую из `bot/utils/tariffs.py`:
```python
tariffs = [
    {
        "code": tariff.code,
        "title": tariff.title,
        "price": f"{tariff.rub_price} ₽",
        "duration": f"{tariff.duration_days} дней доступа",
    }
    for tariff in get_tariffs_list()
]
```

Кэширования нет — каждый запрос загружает актуальные значения из `config`.

### Бот

Бот читает тарифы напрямую из `config` при каждом обращении. Кэширования нет.

## Где используются тарифы

| Компонент | Файл | Как используется |
|-----------|------|-----------------|
| **Бот** | `bot/handlers/tariffs.py` | Отображение тарифов пользователю |
| **Бот** | `bot/payment_flow.py` | Расчёт цены при оплате |
| **Бот** | `bot/platega_flow.py` | Создание платежа с `list_price_amount` |
| **Бот** | `bot/crypto_pay.py` | Создание Crypto Pay инвойса |
| **Dashboard** | `dashboard/finance.py` | Финансовые отчёты |
| **Dashboard** | `dashboard/services.py` | Runtime-обновление тарифов |
| **Landing** | `landing/main.py` | Отображение тарифов на публичной странице |

---

## Влияние на существующие платежи

Изменение тарифов **НЕ влияет** на уже совершённые платежи:
- Существующие `PaymentRecord` хранят `list_price_amount` и `amount` как snapshot
- Подписки пользователей хранят `subscription_expires_at` — фиксированную дату

Новые платежи будут использовать новые цены.

## Слоты устройств

Дополнительные слоты устройств имеют отдельную цену:

```env
DEVICE_SLOT_UNIT_PRICE_RUB=49
DEVICE_SLOT_MAX_EXTRA_SLOTS=5
```

Эти значения также в `bot/config.py` и не связаны с тарифами подписки.
