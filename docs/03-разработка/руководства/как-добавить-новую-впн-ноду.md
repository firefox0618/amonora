# Как добавить новую VPN-ноду

## Обзор

Amonora поддерживает два типа VPN-провайдеров:

| Тип | Provider Type | Протоколы | Описание |
|-----|--------------|-----------|----------|
| **3x-ui** | `xui` | VLESS, Trojan | Панель управления 3x-ui (MHSanaei) |
| **Xray Core** | `xray_core` | VLESS | Прямое управление через SSH (Denmark) |

Решение о типе провайдера принимается по стране через `bot/utils/regions.py`:
```python
ACTIVE_XUI_REGION_CODES = ("de",)      # 3x-ui панели
ACTIVE_XRAY_REGION_CODES = ("dk",)     # Xray Core через SSH
RETIRED_REGION_CODES = ("ee",)         # Выведен из эксплуатации
```

## Сценарий 1: Добавление 3x-ui ноды (новый регион)

### Шаг 1: Подготовка сервера

На целевом сервере:
1. Установи 3x-ui панель (MHSanaei fork):
   ```bash
   bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
   ```
2. Настрой inbound для VLESS на порту 443:
   - Protocol: `vless`
   - Port: `443`
   - Stream: Reality + XHTTP
   - Reality: сгенерируй keypair

3. (Опционально) Настрой inbound для Trojan на порту 8443:
   - Protocol: `trojan`
   - Port: `8443`

4. Убедись что панель доступна:
   ```bash
   curl -v http://127.0.0.1:12053/login
   ```

### Шаг 2: Настройка API доступа

Запиши учётные данные:
- URL панели (внутренний): `http://127.0.0.1:12053`
- Username: `admin`
- Password: `secure_password`

Проверь доступ с хоста бота:
```bash
curl -X POST http://<server-ip>:12053/login \
  -d "username=admin&password=secure_password" \
  -v
# Ожидается: HTTP 200 с session cookie
```

### Шаг 3: Добавление конфигурации

Открой `bot/config.py` — здесь уже описаны все поля Config dataclass.

Добавь в `.env`:

```env
# URL панели для нового региона (например, FR для Франции)
XUI_URL_FR=http://fr-server:12053

# Если нужны отдельные креды для этого региона:
XUI_USERNAME_FR=admin_fr
XUI_PASSWORD_FR=fr_password
```

> **Примечание:** В текущем коде Config dataclass не содержит `xui_url_fr` и `xui_username_fr`. Эти поля нужно добавить в dataclass и в блок инициализации `config = Config(...)`.

Изменения в `bot/config.py`:

1. В dataclass `Config`:
```python
@dataclass
class Config:
    # ... существующие поля ...
    xui_url_fr: str | None = None
    xui_username_fr: str | None = None
    xui_password_fr: str | None = None
```

2. В инициализации:
```python
config = Config(
    # ... существующие параметры ...
    xui_url_fr=get_optional_env("XUI_URL_FR"),
    xui_username_fr=get_optional_env("XUI_USERNAME_FR"),
    xui_password_fr=get_optional_env("XUI_PASSWORD_FR"),
    # ...
)
```

### Шаг 4: Обновление regions/vpn_regions

Открой `bot/utils/regions.py` (требуется уточнение — точный путь зависит от структуры utils):

Добавь новый регион в соответствующие кортежи:

```python
# В vpn_regions.py или аналогичном файле:
ACTIVE_XUI_REGION_CODES = ("de", "fr")  # Добавили "fr"
```

Или если используется функция `get_country_panel_url()`:
```python
def get_country_panel_url(country_code: str | None) -> str | None:
    mapping = {
        "de": config.xui_url_de or config.xui_url,
        "ee": config.xui_url_ee,
        "fr": config.xui_url_fr,  # Добавлено
    }
    return mapping.get(country_code)
```

### Шаг 5: Обновление XUIClient

В `bot/vpn_api.py`, метод `_resolve_country_panel_credentials()` уже поддерживает `ee`. Добавь `fr`:

```python
def _resolve_country_panel_credentials(country_code: str | None) -> tuple[str, str]:
    normalized = str(country_code or "").strip().lower()
    if normalized == "ee":
        return (config.xui_username_ee or config.xui_username,
                config.xui_password_ee or config.xui_password)
    if normalized == "fr":
        return (config.xui_username_fr or config.xui_username,
                config.xui_password_fr or config.xui_password)
    return config.xui_username, config.xui_password
```

И добавь URL mapping в `get_country_panel_url()`.

### Шаг 6: Тестирование подключения

```python
python -c "
import asyncio
from bot.vpn_api import XUIClient

async def test():
    client = XUIClient(country_code='fr')
    login_ok = await client.login()
    print(f'Login OK: {login_ok}')
    if login_ok:
        inbounds = await client.get_inbounds()
        print(f'Inbounds: {len(inbounds.get(\"obj\", []))}')
    await client.close()

asyncio.run(test())
"
```

Ожидаемый вывод:
```
Login OK: True
Inbounds: 2
```

### Шаг 7: Добавление в systemd (опционально)

Если новая нода требует отдельного туннеля или сервиса:

Создай `ops/systemd/amonora-xui-tunnel-fr.service`:
```ini
[Unit]
Description=Amonora XUI Tunnel - France
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/bin/ssh -N -L 12053:127.0.0.1:12053 root@fr-server
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### Шаг 8: Добавление в nginx (опционально)

Если нужен reverse proxy для панели:

`ops/nginx/amonora-fr-web-edge.server.conf`:
```nginx
server {
    listen 443 ssl;
    server_name fr.amonoraconnect.com;

    location / {
        proxy_pass http://127.0.0.1:12053;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### Шаг 9: Проверка через бота

1. Перезапусти бота:
   ```bash
   sudo systemctl restart amonora-control-bot
   # Или: python -m bot.main (dev)
   ```

2. Через control_bot проверь ноду:
   ```
   /nodes
   ```

3. Создай устройство для пользователя через dashboard или бота с регионом `fr`.

4. Проверь что VLESS-ссылка содержит правильный hostname.

## Сценарий 2: Добавление Xray Core ноды (как Denmark)

### Шаг 1: Подготовка сервера

1. Установи Xray Core:
   ```bash
   bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install
   ```

2. Настрой config.json с xhttp inbound:
   ```json
   {
     "inbounds": [{
       "listen": "@xhttp-dk",
       "protocol": "vless",
       "settings": { "clients": [] },
       "streamSettings": {
         "network": "xhttp",
         "xhttpSettings": { ... }
       }
     }]
   }
   ```

3. Создай meta.json с профилями:
   ```json
   {
     "port": 443,
     "reality_server_name": "example.com",
     "reality_short_id": "...",
     "reality_public_key": "...",
     "profiles": {
       "primary": { ... },
       "reserve": { ... }
     }
   }
   ```

### Шаг 2: Настройка SSH доступа

1. Сгенерируй SSH ключ на хосте бота:
   ```bash
   ssh-keygen -t ed25519 -f /opt/amonora_bot/.ssh/xray_dk_key
   ```

2. Добавь публичный ключ на сервер:
   ```bash
   ssh-copy-id -i /opt/amonora_bot/.ssh/xray_dk_key.pub root@dk-server
   ```

3. Добавь сервер в known_hosts:
   ```bash
   ssh-keyscan -p 22 dk-server >> /opt/amonora_bot/.ssh/known_hosts
   ```

### Шаг 3: Добавление конфигурации

В `.env`:

```env
XRAY_CORE_DK_SSH_HOST=dk-server-ip
XRAY_CORE_DK_SSH_PORT=22
XRAY_CORE_DK_SSH_USER=root
XRAY_CORE_DK_SSH_KEY_PATH=/opt/amonora_bot/.ssh/xray_dk_key
XRAY_CORE_DK_SSH_KNOWN_HOSTS=/opt/amonora_bot/.ssh/known_hosts
XRAY_CORE_DK_SSH_TIMEOUT=8
XRAY_CORE_DK_CONFIG_PATH=/usr/local/etc/xray/config.json
XRAY_CORE_DK_META_PATH=/usr/local/etc/xray/amonora_dk_meta.json
```

Для нового региона (например, `NO` для Норвегии) — добавь новые переменные в Config dataclass.

### Шаг 4: Обновление regions

```python
ACTIVE_XRAY_REGION_CODES = ("dk", "no")
```

И добавь провайдер mapping:
```python
def get_country_provider_type(country_code: str | None) -> str:
    if country_code in ACTIVE_XRAY_REGION_CODES:
        return "xray_core"
    if country_code in ACTIVE_XUI_REGION_CODES:
        return "xui"
    return "retired"
```

### Шаг 5: Создание XrayCoreProvisioner

Для нового региона потребуется новый provisioner. В `bot/vpn_provisioning.py`:

```python
class XrayCoreProvisioner_NO(XrayCoreProvisioner):
    def __init__(self, country_code: str):
        # Переопредели SSH-параметры для Норвегии
        self.country_code = country_code
        self.host = config.xray_core_no_ssh_host
        # ... остальные параметры
```

Или (лучше) — параметризуй существующий `XrayCoreProvisioner`.

### Шаг 6: Тестирование

```python
python -c "
import asyncio
from bot.vpn_provisioning import XrayCoreProvisioner

async def test():
    provisioner = XrayCoreProvisioner('dk')
    health = await provisioner.health_check()
    print(f'Health check OK: {health}')
    await provisioner.close()

asyncio.run(test())
"
```

## Архитектура провижининга

```
get_vless_provisioner(country_code, provider_type)
    │
    ├── provider_type == "xray_core" → XrayCoreProvisioner
    │       ├── provision_vless_client() → SSH → Xray config
    │       ├── sync_vless_client() → SSH → обновить expiry
    │       └── build_vless_metadata() → SSH → meta.json → VLESS link
    │
    ├── provider_type == "retired" → RetiredRegionProvisioner (ошибка)
    │
    └── provider_type == "xui" → XUIProvisioner
            ├── provision_vless_client() → 3x-ui API
            ├── sync_vless_client() → 3x-ui API
            └── build_vless_metadata() → 3x-ui API → VLESS link
```

- [ ] `bot/config.py` обновлён (dataclass + инициализация)
