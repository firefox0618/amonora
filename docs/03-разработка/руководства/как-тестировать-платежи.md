# Как тестировать платежи

## Обзор платёжной системы

Amonora поддерживает **5 методов оплаты**:

| Метод | Код | Провайдер | Описание |
|-------|-----|-----------|----------|
| Platega SBP | `sbp_platega` | Platega (method_id=2) | Оплата через СБП (QR-код) |
| Platega Crypto | `crypto_platega` | Platega (method_id=13) | Оплата криптовалютой |
| Crypto Pay | `crypto_pay` | Telegram Crypto Bot | Инвойс в Telegram |
| Telegram Stars | `stars` | Telegram Stars | Внутренняя валюта Telegram |
| Ручной SBP | `manual_sbp` | Ручная обработка | Перевод по реквизитам |
| Ручной Crypto | `manual_crypto` | Ручная обработка | Перевод крипто на кошелёк |

Включение/выключение методов — через ENV-переменные:
```env
ENABLE_PLATEGA_SBP_USER_FLOW=1
ENABLE_PLATEGA_CRYPTO_USER_FLOW=1
ENABLE_CRYPTO_PAY_USER_FLOW=0
ENABLE_MANUAL_SBP_USER_FLOW=0
ENABLE_MANUAL_CRYPTO_USER_FLOW=0
```

## Настройка тестовых токенов Platega

### Шаг 1: Получи тестовые креды

1. Зарегистрируйся на https://app.platega.io
2. В личном кабинете найди:
   - **Merchant ID** (число)
   - **Secret Key** (строка)

### Шаг 2: Добавь в .env

```env
PLATEGA_MERCHANT_ID=12345
PLATEGA_SECRET_KEY=sk_test_...
PLATEGA_BASE_URL=https://app.platega.io
PLATEGA_WEBHOOK_SECRET=whsec_test_...
PLATEGA_CALLBACK_MAX_AGE_SECONDS=1800
```

### Шаг 3: Настрой webhook в Platega

В панели Platega укажи URL webhook:
```
https://www.amonoraconnect.com/webhook/platega
```

Для локального тестирования используй ngrok:
```bash
ngrok http 8000
# Скопируй ngrok URL и укажи в Platega:
# https://xxxx.ngrok-free.app/webhook/platega
```

### Шаг 4: Проверь подключение

```python
python -c "
import asyncio
from bot.platega import PlategaClient

async def test():
    client = PlategaClient()
    print(f'Configured: {client.configured}')
    if client.configured:
        # Попробуй создать тестовый платёж
        payment = await client.create_payment(
            amount_rub=149,
            payment_method_id=PlategaClient.METHOD_SBP_QR,
            description='Test payment',
            payload={'type': 'subscription', 'user_id': 999, 'tariff_code': '1m'},
        )
        print(f'Transaction ID: {payment.transaction_id}')
        print(f'Checkout URL: {payment.checkout_url}')
    await client.close()

asyncio.run(test())
"
```

## Эмуляция успешной оплаты

### Через Platega webhook

1. Создай платёж через бота (выбери тариф → SBP)
2. Получи `external_payment_id` из БД:

```python
python -c "
import asyncio
from bot.db import get_open_payment_record_for_user

async def test():
    record = await get_open_payment_record_for_user(
        user_id=42,
        tariff_code='1m',
        payment_method='sbp_platega',
    )
    if record:
        print(f'Record ID: {record.id}')
        print(f'External ID: {record.external_payment_id}')
        print(f'Status: {record.payment_status}')
    else:
        print('No open payment record found')

asyncio.run(test())
"
```

3. Отправь webhook с CONFIRMED статусом:

```bash
curl -X POST http://127.0.0.1:8000/webhook/platega \
  -H "Content-Type: application/json" \
  -H "X-MerchantId: 12345" \
  -H "X-Secret: sk_test_..." \
  -d '{
    "id": "<external_payment_id>",
    "status": "CONFIRMED",
    "paymentMethod": 2,
    "amount": 149.0,
    "currency": "RUB",
    "payload": "{\"type\":\"subscription\",\"user_id\":42,\"tariff_code\":\"1m\",\"payment_method\":\"sbp_platega\"}",
    "timestamp": "'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"
  }'
```

4. Проверь результат:

```python
python -c "
import asyncio
from bot.db import get_payment_record_by_id

async def test():
    record = await get_payment_record_by_id(123)
    print(f'Status: {record.payment_status}')
    print(f'Confirmed at: {record.confirmed_at}')

asyncio.run(test())
"
```

### Через Dashboard

1. Зайди в Dashboard → Payments
2. Найди платёж со статусом `pending`
3. Нажми "Подтвердить"

Или через API:
```bash
curl -X POST http://127.0.0.1:8088/dashboard/api/v2/payments/<record_id>/confirm \
  -H "Cookie: amonora_dashboard_session=<token>"
```

### Через Control Bot

```
/payments
# Найди платёж → выбери → Confirm
```

## Эмуляция отказа от оплаты

### Через Platega webhook (CANCELED)

```bash
curl -X POST http://127.0.0.1:8000/webhook/platega \
  -H "Content-Type: application/json" \
  -H "X-MerchantId: 12345" \
  -H "X-Secret: sk_test_..." \
  -d '{
    "id": "<external_payment_id>",
    "status": "CANCELED",
    "paymentMethod": 2,
    "amount": 149.0,
    "currency": "RUB",
    "payload": "{\"type\":\"subscription\",\"user_id\":42,\"tariff_code\":\"1m\"}",
    "timestamp": "'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"
  }'
```

### Через Dashboard (Reject)

```bash
curl -X POST http://127.0.0.1:8088/dashboard/api/v2/payments/<record_id>/reject \
  -H "Content-Type: application/json" \
  -H "Cookie: amonora_dashboard_session=<token>" \
  -d '{"reason": "Платёж не получен"}'
```

### Через Control Bot

```
/payments → Найти платёж → Reject → Ввести причину
```

Результат:
- `payment_status` → `expired` или `rejected`
- `rejection_reason` заполняется
- Аналитическое событие `EVENT_PAYMENT_FAILED` создаётся
- В control_bot создаётся событие с `severity=WARNING`

## Ручная обработка платежей (через контрол-бот)

### Включение ручных платежей

```env
ENABLE_MANUAL_SBP_USER_FLOW=1
ENABLE_MANUAL_CRYPTO_USER_FLOW=1
MANUAL_SBP_DETAILS=СБП: +7 XXX XXX-XX-XX (Тинькофф)
MANUAL_CRYPTO_DETAILS=USDT (TRC-20): TXxx...xxx
MANUAL_PAYMENT_REVIEW_HOURS=12
```

### Создание ручного платежа

Пользователь выбирает "Ручной SBP" → получает реквизиты → делает перевод.

### Обработка через Dashboard

1. Dashboard → Payments → Создать платёж вручную:
   ```json
   {
     "user_id": 42,
     "payment_method": "manual_sbp",
     "tariff_code": "3m",
     "payment_status": "pending",
     "reference": "Чек №12345",
     "note": "Перевод от Ивана"
   }
   ```

2. После получения денег — подтвердить:
   ```
   POST /dashboard/api/v2/payments/{record_id}/confirm
   ```

### Обработка через Control Bot

```
/payments → Pending → Выбрать → Confirm
```

### Проверка в БД

```sql
SELECT id, user_id, payment_method, payment_status, amount,
       reference, reviewed_by_actor_name, reviewed_at
FROM payment_records
WHERE payment_method LIKE 'manual%'
ORDER BY created_at DESC
LIMIT 10;
```

## Тестирование Crypto Pay

### Настройка

```env
CRYPTO_PAY_API_TOKEN=crypto_pay_token_...
CRYPTO_PAY_BASE_URL=https://pay.crypt.bot/api
CRYPTO_PAY_WEBHOOK_SECRET=cp_whsec_...
CRYPTO_PAY_ACCEPTED_ASSETS=USDT,TON
CRYPTO_PAY_SWAP_TO=TON
CRYPTO_PAY_INVOICE_EXPIRES_SECONDS=3600
ENABLE_CRYPTO_PAY_USER_FLOW=1
```

### Создание инвойса

```python
python -c "
import asyncio
from bot.crypto_pay import CryptoPayClient

async def test():
    client = CryptoPayClient()
    print(f'Configured: {client.configured}')
    if client.configured:
        invoice = await client.create_invoice(
            amount_rub=149,
            title='Test payment',
            payload={'type': 'subscription', 'user_id': 42, 'tariff_code': '1m'},
        )
        print(f'Invoice ID: {invoice.invoice_id}')
        print(f'Pay URL: {invoice.pay_url}')
        print(f'Status: {invoice.status}')

asyncio.run(test())
"
```

### Верификация webhook подписи

Crypto Pay webhook проверяется в `bot/crypto_pay.py`:

```python
def verify_webhook_signature(self, raw_body: bytes, signature: str | None) -> bool:
    if not self.api_token or not signature:
        return False
    secret = hashlib.sha256(self.api_token.encode("utf-8")).digest()
    expected = hmac.new(secret, raw_body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, signature)
```

### Эмуляция webhook Crypto Pay

```bash
# Получить body и signature от Crypto Pay
# Или создать тестовый инвойс и оплатить через тестовый кошелёк

curl -X POST http://127.0.0.1:8000/webhook/crypto-pay \
  -H "Content-Type: application/json" \
  -H "X-Crypto-Pay-Signature: <signature>" \
  -d '{
    "update_id": 12345,
    "update_type": "invoice_paid",
    "date": "2026-04-14T10:30:00Z",
    "payload": {
      "invoice_id": "<invoice_id>",
      "status": "paid",
      "amount": "149",
      "fiat": "RUB",
      "currency_type": "crypto",
      "asset": "USDT",
      "pay_amount": "15.5",
      "pay_asset": "TON",
      "payload": "{\"type\":\"subscription\",\"user_id\":42,\"tariff_code\":\"1m\"}"
    }
  }'
```

## Тестирование Telegram Stars

Stars интеграция проходит через Telegram Bot API. Для тестирования:

1. Установи `STARS_PROVIDER_TOKEN` (требует уточнения — токен от Telegram)
2. `STARS_CURRENCY=XTR` (по умолчанию)

Тарифы в Stars:
```env
TARIFF_1M_STARS=299
TARIFF_3M_STARS=799
TARIFF_6M_STARS=1499
TARIFF_12M_STARS=2799
```

Тестирование Stars ограничено — требует реального бота в Telegram с подключённым Stars провайдером.

## Где смотреть результаты в БД

### Таблица payment_records

```sql
-- Последние платежи
SELECT
    id,
    user_id,
    payment_method,
    tariff_code,
    payment_status,
    amount,
    list_price_amount,
    balance_applied_amount,
    currency,
    external_payment_id,
    reference,
    created_at,
    confirmed_at,
    reviewed_by_actor_name,
    rejection_reason
FROM payment_records
ORDER BY created_at DESC
LIMIT 20;
```

### Статусы платежей

| Статус | Описание |
|--------|----------|
| `pending` | Ожидает оплаты |
| `confirmed` | Подтверждён, доступ активирован |
| `expired` | Истёк / отменён |
| `rejected` | Отклонён администратором |

### Проверка активации подписки

```sql
-- Проверить что подписка активирована
SELECT
    u.id,
    u.telegram_id,
    u.subscription_status,
    u.subscription_started_at,
    u.subscription_expires_at,
    u.subscription_source
FROM users u
WHERE u.id = 42;
```

### Финансовые записи

```sql
-- Финансовые записи связанные с платежами
SELECT
    fe.id,
    fe.entry_type,
    fe.category,
    fe.amount,
    fe.status,
    fe.source_type,
    fe.source_id,
    fe.approved_at
FROM finance_entries fe
WHERE fe.source_type = 'payment_record'
ORDER BY fe.created_at DESC
LIMIT 10;
```

### Баланс пользователя

```sql
SELECT
    id,
    telegram_id,
    balance_rub,
    balance_reserved_rub,
    referral_earned_total_rub
FROM users
WHERE id = 42;
```

### События баланса

```sql
SELECT
    id,
    user_id,
    amount,
    direction,   -- 'credit' или 'debit'
    reason,
    reference_type,
    reference_id,
    note,
    created_at
FROM user_balance_events
WHERE user_id = 42
ORDER BY created_at DESC
LIMIT 20;
```

### Аналитические события

```sql
-- Аналитика платежей
SELECT
    event_name,
    occurred_at,
    user_id,
    payment_record_id,
    tariff_code,
    payment_method,
    payload
FROM analytics_events
WHERE event_name IN ('payment_success', 'payment_failed')
ORDER BY occurred_at DESC
LIMIT 20;
```

### Control Bot события

```sql
-- События control_bot о платежах
SELECT
    id,
    category,
    severity,
    event_type,
    title,
    message,
    entity_type,
    entity_id,
    created_at
FROM control_notification_events
WHERE category = 'payments'
ORDER BY created_at DESC
LIMIT 10;
```
