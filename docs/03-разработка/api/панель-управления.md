# API панели управления Amonora Dashboard

## Общая информация

- **Базовый URL:** `http://127.0.0.1:8088` (по умолчанию)
- **Фреймворк:** FastAPI 0.122.0
- **ASGI-сервер:** Uvicorn 0.38.0
- **Порт:** настраивается через `DASHBOARD_PORT` (по умолчанию 8088)

---

## Аутентификация

Dashboard использует **двухфакторную аутентификацию** через Telegram:

1. **POST /login** — `username` + `password` → код отправляется в Telegram
2. **POST /verify** — `username` + `code` → устанавливается `httponly` cookie сессии

Все эндпоинты `/dashboard/*` требуют cookie-токен сессии. API v2 эндпоинты (`/dashboard/api/v2/*`) возвращают JSON.

### Роли администраторов

| Роль | Описание |
|------|----------|
| `owner` | Полный доступ, включая удаление пользователей |
| `tech` | Управление серверами, пользователями, финансами |
| `support` | Обращения в поддержку, просмотр пользователей |

Разрешения проверяются через `role_has_permission(role, permission)`:
- `manage_users` — управление пользователями
- `manage_servers` — управление серверами
- `manage_server_actions` — действия на серверах
- `clear_access` — отключение доступа
- `run_sync` — синхронизация VPN
- `run_deep_repair` — глубокий ремонт

## Эндпоинты аутентификации

### POST /login

Вход в dashboard. Отправляет код подтверждения в Telegram.

**Request (form-data):**
```
username=rudolf
password=secret
```

**Response:** Redirect на `/verify?username=...` или `/login?error=...`

### POST /verify

Верификация кода из Telegram.

**Request (form-data):**
```
username=rudolf
code=123456
```

**Response:** Redirect на `/dashboard/overview` с установкой cookie-сессии.

### POST /logout

Выход из dashboard.

**Response:** Redirect на `/login` с удалением cookie.

## API v2 эндпоинты

Все эндпоинты возвращают JSON в формате:
```json
{
  "ok": true,
  "data": { ... },
  "notice": "Сообщение"
}
```

Ошибка:
```json
{
  "ok": false,
  "error": "Описание ошибки"
}
```

## Кэширование

API v2 эндпоинты используют read-through кэш:

```python
_V2_READ_CACHE: dict[str, tuple[float, object]] = {}
```

| Payload | TTL (сек) |
|---------|-----------|
| overview | 45 |
| servers | 45 |
| traffic | 45 |
| notifications | 30 |
| audit | 30 |
| users | 20 |
| user-detail | 15 |
| search | 10 |

Кэш инвалидируется после write-операций через `_invalidate_v2_cache()`.
