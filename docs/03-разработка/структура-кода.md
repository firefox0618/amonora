## Ключевые компоненты

### Модели данных — `backend/core/models.py`

**Правило: `backend/core/models.py` — единственный источник истины для всех ORM-моделей.**

Все компоненты (bot, dashboard, landing, support_bot, control_bot) импортируют модели отсюда:

```python
from backend.core.models import User, VpnClient, PaymentRecord, SupportTicket
```

Основные модели:

| Модель | Таблица | Описание |
|--------|---------|----------|
| `User` | `users` | Пользователи, подписки, баланс, рефералы |
| `VpnClient` | `vpn_clients` | VPN-клиенты (VLESS, Trojan) |
| `VpnClientActivation` | `vpn_client_activations` | Активации устройств с fingerprint |
| `PublicSubscriptionLink` | `public_subscription_links` | Публичные ссылки подписок |
| `PublicSubscriptionRoute` | `public_subscription_routes` | Маршруты публичных подписок по странам |
| `DeviceSlotEntitlement` | `device_slot_entitlements` | Права на слоты устройств |
| `VpnRepairEvent` | `vpn_repair_events` | События восстановления VPN |
| `DeviceCompensationJob` | `device_compensation_jobs` | Фоновые задачи компенсации устройств |
| `UserDeletionJob` | `user_deletion_jobs` | Фоновые задачи удаления пользователей |
| `SupportTicket` | `support_tickets` | Обращения в поддержку |
| `SupportTicketMessage` | `support_ticket_messages` | Сообщения в тикетах |
| `UserBalanceEvent` | `user_balance_events` | События баланса пользователя |
| `Referral` | `referrals` | Реферальные связи |
| `ReferralReward` | `referral_rewards` | Реферальные вознаграждения |
| `PromoCode` | `promo_codes` | Промокоды |
| `PromoCodeRedemption` | `promo_code_redemptions` | Использования промокодов |
| `ControlNotificationEvent` | `control_notification_events` | События уведомлений control_bot |
| `ControlBroadcastCampaign` | `control_broadcast_campaigns` | Рассылки |

Модели dashboard (`dashboard/models.py`) дополняют:
- `DashboardAdmin` — администраторы dashboard
- `DashboardSession` — сессии входа
- `PaymentRecord` — записи платежей

- и другие

### База данных — `backend/core/database.py`

```python
from backend.core.database import Base, engine, async_session
```

- `Base` — declarative base для SQLAlchemy
- `engine` — AsyncEngine (postgresql+asyncpg)
- `async_session` — фабрика AsyncSession

### Обработчики бота — `bot/handlers/`

Каждый файл — независимый aiogram `Router`:

| Файл | Роутер | Описание |
|------|--------|----------|
| `start.py` | `start_router` | `/start`, триал, проверка подписки на канал |
| `cabinet.py` | `cabinet_router` | Личный кабинет, баланс, статус подписки |
| `devices.py` | `devices_router` | Создание/удаление устройств, VLESS-ссылки |
| `protocol.py` | `protocol_router` | Выбор протокола (VLESS/Trojan) |
| `tariffs.py` | `tariffs_router` | Просмотр и покупка тарифов |
| `referrals.py` | `referrals_router` | Реферальная программа, бонусы |
| `info.py` | `info_router` | Информационные разделы |
| `support.py` | `support_router` | Создание обращения в поддержку |

Регистрация роутеров в `bot/main.py`:

```python
dp.include_router(v2_router)       # test_bot
dp.include_router(start_router)
dp.include_router(cabinet_router)
dp.include_router(devices_router)
dp.include_router(protocol_router)
dp.include_router(info_router)
dp.include_router(referrals_router)
dp.include_router(support_router)
dp.include_router(tariffs_router)
```

## Правило: backend/core/ — единственный источник истины для моделей

**Запрещено:**
- Определять ORM-модели в `bot/`, `dashboard/`, `landing/` или любом другом компоненте.
- Создавать дубликаты моделей в разных компонентах.

**Разрешено:**
- `dashboard/models.py` определяет модели, специфичные для dashboard (администраторы, сессии dashboard).
- `backend/core/models.py` содержит все доменные модели (User, VpnClient, PaymentRecord, SupportTicket и т.д.).

Импорт во всех компонентах:
```python
from backend.core.models import User, VpnClient, PaymentRecord
from backend.core.database import async_session, Base
from backend.core.schema import ensure_schema
from backend.core.analytics import EVENT_PAYMENT_SUCCESS, safe_emit_analytics_event
```
