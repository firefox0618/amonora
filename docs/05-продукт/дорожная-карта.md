# Дорожная карта

## Ближайшие задачи (текущие)

### Продукт

- **Стабилизация trial-воронки** — оптимизация конверсии из trial в платную подписку через триггерные сообщения
  - Триггеры уже настроены: 2h, 24h, 6h до конца, 3d после
  - Требуется мониторинг эффективности и A/B-тестирование текстов

- **Улучшение payment flow** — поддержка множественных методов оплаты
  - Platega (СБП/крипто) — работает
  - Telegram Stars — работает
  - Ручные оплаты (СБП/крипто) — работают
  - Баланс (RUB) — работает

- **User-level subscription page** (`client.amonoraconnect.com/<token>`)
  - Публикация статуса подписки, QR, install-flow
  - Happ feed integration
  - Server-side bind к слотам

### Инфраструктура

- **Мониторинг серверов** — control_bot алерты + dashboard health checks
  - Мониторинг нод через systemd watchdog
  - ~~Grafana~~ удалена — мониторинг через control_bot + dashboard

- **VPN-ноды** — три региона
  - Германия (3x-ui) — основной
  - Дания (Xray core) — мобильный режим
  - Эстония (резерв) — fallback

### Безопасность

- **Anti-sharing система** — контроль шаринга ключей
  - `vpn_antisharing_lease_seconds` — аренда (по умолчанию 180 сек)
  - `vpn_antisharing_soft_limit_enabled` — мягкий лимит включён
  - `VPN_MAX_DEVICES_PER_KEY` — максимум устройств

- **Synthetic users** — фильтрация синтетических пользователей из аналитики
  - `is_synthetic` флаг в модели User
  - Bridge-access users не попадают в обычные агрегаты

## Технические долги

### Dashboard V1 vs V2

- `dashboard/` — текущая панель управления (Legacy + Active)
- `dashboard_v2` — новая версия (в разработке)
- **Риск:** `dashboard/` не полностью legacy — содержит активные backend/API responsibilities
- **Правило:** не удалять старые UI-флоу из `dashboard/` без подтверждения покрытия в `dashboard_v2`

**Источник:** `AGENTS.md`

### Переход от файлов к БД для поддержки

- Support_bot мигрировал с JSON-файлов на БД (`SupportTicket`, `SupportTicketMessage`)
- Legacy-файлы архивируются с суффиксом `.migrated-*`

**Источник:** `support_bot/storage.py` — `_migrate_legacy_file()`

### Promo-система

- Promo-акции были удалены (текущий `promo_active()` возвращает `False`)
- Старые промо-тексты в триггерах сохранены как stale-копии
- Тест `test_promo_copy_consistency.py` проверяет консистентность

**Источник:** `bot/utils/tariffs.py` — `promo_active()`, `tests/test_promo_copy_consistency.py`

### Реферальный баланс (миграция)

- Существовала миграция `referral_balance_migrated_at` для исторических бонусов
- Legacy cutoff дата для миграции
- Backfill для пропущенных начислений

**Источник:** `tests/test_referral_balance.py` — `REFERRAL_BALANCE_LEGACY_CUTOFF`
