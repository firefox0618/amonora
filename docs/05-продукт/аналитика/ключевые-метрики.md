# Ключевые метрики

## Обзор

Система аналитики Amonora собирает метрики через события (`AnalyticsEvent`) и агрегирует их в rollup-таблицы (`analytics_daily_*`). Метрики доступны через **dashboard API** и напрямую из БД.

**Источник:** `backend/core/analytics.py`, `ops/analytics_refresh.py`, `dashboard/analytics.py`

---

## Список KPI

### MAU (Monthly Active Users)

**Описание:** Количество пользователей с активной подпиской за месяц.

**Где считается:** `analytics_runtime_status` → `growth_active_users`

**Формула:** Подсчёт пользователей со статусом `paid_active` или `trial_active`.

### LTV (Lifetime Value)

**Описание:** Средний доход от одного платящего пользователя за всё время.

**Формула:** `total_revenue / unique_paying_users`

### Churn Rate — Отток

**Описание:** Процент пользователей, у которых истекла подписка и не продлили.

**Где считается:** `analytics_cohort_retention` (cohorted by signup date)

### Конверсия в оплату

**Описание:** Процент пользователей, перешедших от триала к платной подписке.

**Формула:** `paying_users / total_trial_users * 100`

### Выручка

**Описание:** Суммарный доход за период (RUB + Stars в RUB-эквиваленте).

**Где считается:** `analytics_daily_revenue`

---

## Воронка конверсии

**Этапы:**

1. **start** — нажал `/start`
2. **agreement** — принял соглашение
3. **trial** — активировал триал
4. **paying** — оплатил подписку
5. **renewed** — продлил подписку

**Где считается:** `analytics_daily_stage_counts`, `analytics_daily_stage_segments`

---

## Источники данных

### Таблицы БД

| Таблица | Что хранит |
|---------|-----------|
| `analytics_daily_stage_counts` | Ежедневные этапы воронки |
| `analytics_daily_revenue` | Ежедневная выручка |
| `analytics_daily_connection` | Статистика подключений |
| `analytics_daily_payment_failure_reasons` | Причины сбоев оплаты |
| `analytics_cohort_retention` | Когортный анализ (0, 1, 3, 7, 14, 30, 60, 90 дней) |
| `analytics_user_attribution` | Источники привлечения пользователей |
| `analytics_events` | Сырые события воронки |

### Обновление данных

Rollup-таблицы обновляются через `ops/analytics_refresh.py`. Запускается по cron-таймеру.

## Где смотреть

- **Dashboard** — основные метрики доступны через админку
- **БД напрямую** — через `psql -h 127.0.0.1 -U amonora -d amonora`
- **Dashboard API** — `GET /dashboard/api/v2/analytics/*`
