# Воронка конверсии

## Этапы воронки

Воронка Amonora отслеывает путь пользователя от первого контакта до продления подписки.

**Источник:** `backend/core/analytics.py` — `ROLLUP_STAGE_EVENTS`, `dashboard/analytics.py` — funnel queries

> **Примечание:** Grafana удалена. `ops/grafana/build_suite.py` больше не используется — запросы к воронке выполняются через dashboard.

### Полная воронка

| # | Этап | Событие | Описание |
|---|------|---------|----------|
| 1 | Переход по ссылке | `link_touched` | Пользователь перешёл по реферальной ссылке или ссылке на канал |
| 2 | Старт в боте | `bot_start` | Пользователь нажал `/start` |
| 3 | Подписка на канал | `channel_membership_confirmed` | Подтвердил подписку на `@amonora_new` |
| 4 | Начало подключения | `connection_started` | Начал создание устройства |
| 5 | Готов к подключению | `connection_ready` | Техническая настройка завершена |
| 6 | Ключ выдан | `config_issued` | Получил VPN-ключ |
| 7 | Оплата начата | `subscription_payment_started` | Начал процесс оплаты |
| 8 | Оплата успешна | `subscription_activated` / `subscription_renewed` | Подписка активирована |
| 9 | Первое подключение | `first_connection_success` | Успешно подключился через VPN-клиент |
| 10 | Продление | `subscription_renewed` | Продлил подписку |

### Укороченная воронка (executive-срез)

Для быстрого просмотра используются ключевые этапы:

1. Переход по ссылке
2. Старт в боте
3. Подписка на канал
4. Ключ выдан
5. Оплата успешна
6. Первое подключение
7. Продление

**Источник:** `ops/grafana/build_suite.py` — `funnel_cte()` — `step_order IN (1, 2, 3, 6, 8, 9, 10)`

## Источники трафика

Воронка сегментируется по источникам:

| Источник | Значение `source_type` | Описание |
|----------|----------------------|----------|
| Органика | `organic_bot` | Прямой заход в бот |
| Канал | `channel_post` | Переход из поста канала |
| Реферал | `organic_bot` + `ref_*` | По реферальной ссылке |

**Атрибуция:**
- `first` — первый источник (первоначальный)
- `last` — последний источник (текущий)

**Источник:** `backend/core/analytics.py` — `SOURCE_TYPE_ORGANIC`, `SOURCE_TYPE_CHANNEL_POST`

## Типичные значения конверсии

**Требуется уточнение:** конкретные значения конверсии зависят от реальных данных и не зафиксированы в коде.

Однако структура воронки позволяет вычислить:

| Метрика | Формула |
|---------|---------|
| Конверсия старт → оплата | `subscription_activated / bot_start * 100` |
| Конверсия оплата → подключение | `first_connection_success / subscription_activated * 100` |
| Конверсия ключ → оплата | `subscription_activated / config_issued * 100` |
| Доля продлений | `renewals / (first_payments + renewals) * 100` |

В Grafana эти метрики отображаются в процентах с цветовой индикацией порогов.

## Разрыв оплата → подключение

Отдельная метрика отслеживает пользователей, которые оплатили, но не подключились:

```
Разрыв = paid_users - connected_users
```

Это важно для выявления проблем в процессе выдачи ключей или подключения.

**Источник:** `ops/grafana/build_suite.py` — панель «Разрыв оплата → подключение»

## Целостность данных

Метрика `integrity_ratio` отслеживает полноту данных атрибуции:

```
Целостность = (события с корректным source_key) / (все события) * 100
```

Цель: близкая к 100%.

**Источник:** `ops/grafana/build_suite.py` — панель «Целостность атрибуции»

## Как улучшить конверсию

### На основе структуры воронки

| Потеря на этапе | Возможная причина | Инструменты улучшения |
|----------------|-------------------|----------------------|
| Старт → Подписка на канал | Не хотят подписываться | Улучшить ценность подписки в тексте |
| Подписка → Trial | Не понимают trial | Упростить активацию |
| Trial → Ключ | Сложная настройка | Follow-up сообщения через 2ч, 24ч |
| Ключ → Оплата | Нет денег / не готовы | Промокоды, напоминания |
| Оплата → Подключение | Технические проблемы | Авто-ремонт (`vpn_repair_needed`) |
| Подключение → Продление | Не понравилось | Улучшение качества VPN, напоминания |

### Триггерные сообщения (Control Bot)

Система уже использует автоматические напоминания:

| Триггер | Когда | Действие |
|---------|-------|----------|
| `trial_active_2h` | Через 2ч после старта активного trial | CTA → оплата |
| `trial_low_2h` | Через 2ч после старта неактивного trial | CTA → настройка |
| `trial_active_24h` | Через 24ч активного trial | CTA → оплата |
| `trial_low_24h` | Через 24ч неактивного trial | CTA → настройка |
| `trial_final_6h` | За 6ч до конца trial | CTA → оплата |
| `trial_expired_3d` | Через 3 дня после окончания | CTA → оплата |
| `start_no_action_1h` | Через 1ч без действий | CTA → trial |
| `start_no_action_3h` | Через 3ч без действий | CTA → поддержка |
| `subscription_expires_3d` | За 3 дня до окончания | CTA → продление |
| `subscription_expires_today` | В день окончания | CTA → продление |

**Источник:** `control_bot/storage.py` — `TRIGGER_DEFAULTS`
