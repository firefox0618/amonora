# Воронка конверсии

> Воронка описывает путь пользователя от первого контакта до продления подписки. Данные хранятся в `analytics_daily_stage_counts` и `analytics_events`.

## Этапы воронки

| Этап | Ключ | Описание |
|------|------|----------|
| Вход | `start` | Нажал `/start` |
| Соглашение | `agreement` | Принял пользовательское соглашение |
| Триал | `trial` | Активировал пробный период |
| Оплата | `paying` | Оплатил подписку |
| Продление | `renewed` | Продлил подписку |

## Ключевые метрики воронки

| Метрика | Описание |
|---------|----------|
| **start → trial** | Конверсия в активацию триала |
| **trial → paying** | Конверсия триала в оплату |
| **paying → renewed** | Процент продлений |
| **Оплата → подключение** | Разрыв между оплатой и первым подключением к VPN |

## Где данные

| Источник | Таблица |
|----------|---------|
| Ежедневные этапы | `analytics_daily_stage_counts` |
| Сегменты этапов | `analytics_daily_stage_segments` |
| Сырые события | `analytics_events` |

Метрики доступны через **dashboard API** (`/dashboard/api/v2/analytics/*`) и напрямую из БД.

## Обновление

Rollup-таблицы обновляются через `ops/analytics_refresh.py` по cron-таймеру. Целостность данных проверяется через `analytics_daily_attribution_integrity`.
