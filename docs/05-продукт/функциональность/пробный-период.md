# Пробный период (Trial)

## Длительность

Пробный период составляет **3 дня** по умолчанию.

Задаётся через переменную окружения `TRIAL_DAYS` (значение по умолчанию: 3).

**Источник:** `bot/config.py` — `trial_days=int(get_env("TRIAL_DAYS", "3"))`

## Условия получения

### Один раз навсегда

Trial выдаётся **один раз на аккаунт**. Поле `User.trial_used` фиксирует факт использования. После окончания trial-периода повторная активация невозможна.

**Источник:** `backend/core/models.py` — `User.trial_used: Mapped[bool]`

### Требование подписки на канал

Для активации trial пользователь **обязан быть подписанным** на канал проекта (`@amonora_new`).

Flow активации:
1. Пользователь нажимает `/start`
2. Если не подписан — показывается кнопка «Подписаться на канал» и «Подписался»
3. После нажатия «Подписался» бот проверяет подписку через Telegram API
4. Если подписка подтверждена — trial активируется

**Источник:** `bot/handlers/start.py` — `_handle_start_flow()`, `TRIAL_SUBSCRIPTION_CONFIRMED_CALLBACK`

### Пауза trial при отписке

Если пользователь отписывается от канала во время trial:
- Trial **приостанавливается** (не прекращается)
- Оставшееся время сохраняется
- При повторной подписке trial **возобновляется** с оставшегося времени
- Новый trial не выдаётся

**Источник:** `bot/handlers/start.py` — `trial_is_paused_by_channel_from_user()`, `resume_trial_after_channel_resubscribe()`

## Что доступно во время trial

| Параметр | Значение |
|----------|----------|
| Устройства | Базовый лимит — 3 устройства |
| Страны | Германия (основная), Дания (Xray), Эстония (резерв) |
| Протоколы | VLESS (по умолчанию), Trojan |
| Трафик | Без лимитов |
| Доп. слоты | **Недоступны** (только для платных подписок) |
| Реферальная система | Доступна |
| Промокоды | Доступны |

Режимы подключения:
- **Стабильный** — VLESS + Reality + TCP
- **Мобильный** — VLESS + Reality + XHTTP (Дания)
- **Резерв** — Эстония

**Источник:** `documentation/FEATURES.md`, `bot/utils/device_slots.py`

## Активность во время trial

Система отслеживает активность trial-пользователя:

| Уровень | Описание |
|---------|----------|
| `low` | Пользователь не дошёл до технической настройки |
| `active` | Пользователь начал настройку (создал устройство, получил ключ) |

Поля модели:
- `trial_activity_level` — текущий уровень активности
- `trial_engaged_at` — время первого технического взаимодействия
- `trial_started_at` — время начала trial
- `trial_expires_at` — время окончания trial

**Источник:** `backend/core/models.py`, `tests/test_trial_funnel.py`

## Конверсия Trial → Paid

Метрика конверсии рассчитывается в дашборде:

```
trial_to_paid_percent = (пользователи с paid_active И trial_used) / (все пользователи с trial_used) * 100
```

Дашборд показывает:
- Количество trial-пользователей за период
- Процент перехода в платную подписку
- Сравнение с предыдущим периодом

**Источник:** `dashboard/analytics.py` — `_calculate_user_metrics()`

## Блокировка после окончания trial

После истечения `trial_expires_at`:

1. Статус доступа меняется на `inactive` / `trial_expired`
2. VPN-ключи деактивируются на стороне xui
3. Пользователь видит сообщение «Доступ истёк» с предложением выбрать тариф
4. Устройства больше не выдаются
5. Пользователь может оплатить тариф для восстановления доступа

Напоминания перед окончанием trial (через control_bot):
- За 1 день (`trial_ends_1d`)
- В день окончания (`trial_ends_today`)
- Через 2 часа после старта (`trial_active_2h`, `trial_low_2h`)
- Через 24 часа (`trial_active_24h`, `trial_low_24h`)
- За 6 часов до конца (`trial_final_6h`)
- Через 3 дня после окончания (`trial_expired_3d`)

**Источник:** `control_bot/storage.py` — `TRIGGER_DEFAULTS`, `bot/utils/texts.py` — `access_expired_text()`
