# Промокоды

## Типы промокодов

Система поддерживает три типа промокодов:

| Тип | Код в системе | Описание |
|-----|--------------|----------|
| **Скидка в процентах** | `discount_percent` | Процентная скидка на следующую оплату (макс. 95%) |
| **Дни доступа** | `days_credit` | Бесплатное продление подписки на N дней |
| **Подарочный** | `gift_days` | Одноразовая подарочная подписка (генерируется автоматически) |

**Источник:** `backend/core/promo_codes.py` — `PROMO_KIND_DISCOUNT_PERCENT`, `PROMO_KIND_DAYS_CREDIT`, `PROMO_KIND_GIFT_DAYS`

## Модель промокода

Таблица `promo_codes` содержит:

| Поле | Описание |
|------|----------|
| `code` | Уникальный код (до 64 символов, UPPER CASE + цифры + `-`) |
| `kind` | Тип промокода |
| `title` | Отображаемое название |
| `description` | Описание |
| `discount_percent` | Процент скидки (для `discount_percent`) |
| `grant_days` | Количество дней (для `days_credit`, `gift_days`) |
| `max_redemptions` | Максимум активаций (по умолчанию 1) |
| `redeemed_count` | Текущее количество активаций |
| `status` | `active`, `inactive`, `exhausted` |
| `created_by_admin_id` | Админ, создавший промокод |
| `buyer_user_id` | Пользователь, купивший подарочный промокод |
| `payment_record_id` | Привязка к платежу (для подарочных) |
| `expires_at` | Срок действия (опционально) |

Таблица `promo_code_redemptions` фиксирует каждое использование:

| Поле | Описание |
|------|----------|
| `promo_code_id` | Ссылка на промокод |
| `user_id` | Кто активировал |
| `status` | `pending_discount`, `applied`, `consumed` |
| `discount_percent` | Применённая скидка |
| `granted_days` | Начисленные дни |
| `applied_payment_record_id` | К какому платежу применена скидка |
| `redeemed_at` | Время активации |
| `applied_at` | Время применения (для скидок — при оплате) |

**Источник:** `backend/core/models.py` — `PromoCode`, `PromoCodeRedemption`

## Как создать промокод

### Через панель управления (Dashboard)

Администратор создаёт промокод через dashboard:
1. Раздел «Промокоды»
2. Указывает тип, значение, срок действия, лимит активаций
3. Система генерирует уникальный код (если не указан вручную)

### Программное создание

Функция `create_promo_code()` принимает:
- `code` — код (или генерируется автоматически с префиксом `AMONORA-XXXXXXXX`)
- `kind` — тип
- `discount_percent` / `grant_days` — значение
- `max_redemptions` — лимит использований
- `expires_at` — срок действия
- `created_by_admin_id` — кто создал

### Подарочные промокоды

Генерируются автоматически при оплате подписки через `create_gift_promo_code_for_payment()`:
- Привязываются к `payment_record_id`
- Тип: `gift_days`
- Одноразовые (`max_redemptions = 1`)
- Код генерируется автоматически

**Источник:** `backend/core/promo_codes.py` — `create_promo_code()`, `create_gift_promo_code_for_payment()`

## Как применяется промокод

### Процентная скидка (`discount_percent`)

1. Пользователь вводит промокод
2. Создаётся запись `PromoCodeRedemption` со статусом `pending_discount`
3. При следующей оплате подписки скидка применяется автоматически:
   - `apply_discount_percent(amount, percent)` — расчёт итоговой суммы
   - Редemption получает статус `consumed`
   - Привязывается к `payment_record_id`

**Ограничение:** нельзя иметь две активные процентные скидки одновременно. Нужно сначала использовать текущую.

### Дни доступа (`days_credit`, `gift_days`)

1. Пользователь вводит промокод
2. Подписка немедленно продлевается на `grant_days`
3. Запись получает статус `applied`
4. VPN-доступ синхронизируется автоматически

Разница между `days_credit` и `gift_days`:
- `days_credit` — администратор выдаёт дни доступа
- `gift_days` — подарочный код, привязанный к платежу

**Источник:** `backend/core/promo_codes.py` — `redeem_promo_code_for_user()`

## Ограничения

| Ограничение | Значение |
|-------------|----------|
| Максимум использований | Настраивается (`max_redemptions`), по умолчанию 1 |
| Максимальная скидка | 95% |
| Уникальность кода | Один код — одна запись |
| Повторная активация | Запрещена (один пользователь — один промокод) |
| Срок действия | Опциональный (`expires_at`); после истечения — статус `exhausted` |
| Истёкший промокод | Автоматически помечается как `exhausted` |
| Исчерпанный лимит | Автоматически помечается как `exhausted` |

## Комбинирование с рефералами

Промокоды и реферальные бонусы **работают независимо**:

- **Реферальные бонусы** — зачисляются на баланс (RUB), используются при RUB-оплате
- **Промокод-скидка** — применяется к сумме платежа в процентах
- **Промокод-дни** — добавляют дни к подписке, не влияют на оплату

Возможные сценарии:
1. Реферальный бонус на балансе + промокод-скидка = максимальная экономия
2. Реферальный бонус покрывает остаток после применения скидки

**Источник:** логика в `bot/handlers/tariffs.py` — баланс и промо обрабатываются раздельно

## Примеры использования

### Сценарий 1: Процентная скидка

```
Пользователь вводит промокод SALE20
→ discount_percent = 20
→ pending_discount создан

Пользователь покупает тариф 3 месяца (399 ₽)
→ скидка 20% = 80 ₽
→ к оплате: 319 ₽
→ редemption статус: consumed
```

### Сценарий 2: Дни доступа

```
Пользователь вводит промокод BONUS7
→ grant_days = 7
→ подписка продлена на 7 дней
→ VPN синхронизирован
→ редemption статус: applied
```

### Сценарий 3: Подарочный код

```
Пользователь A покупает подписку
→ автоматически создаётся gift промокод AMONORA-ABCD1234
→ grant_days = 30 (равно тарифу)
→ Пользователь A отправляет код другу B
→ Пользователь B активирует код
→ Подписка B продлена на 30 дней
→ Код больше недействителен (max_redemptions = 1)
```

## Нормализация кода

При вводе промокод нормализуется:
- Приводится к UPPER CASE
- Удаляются все символы кроме `A-Z`, `0-9`, `-`, `_`
- Обрезается до 64 символов

**Источник:** `backend/core/promo_codes.py` — `normalize_promo_code()`
