# Тарифы и цены

## Обзор

Amonora предлагает VPN-доступ через Telegram-бота с подписочной моделью. Оплата возможна в рублях (RUB) и Telegram Stars (XTR).

## Таблица тарифов

| Код | Название | Срок (дней) | Цена RUB | Цена Stars |
|-----|----------|-------------|----------|------------|
| `1m` | 1 месяц | 30 | 149 ₽ | 299 ★ |
| `3m` | 3 месяца | 90 | 399 ₽ | 799 ★ |
| `6m` | 6 месяцев | 180 | 749 ₽ | 1499 ★ |
| `12m` | 12 месяцев | 365 | 1390 ₽ | 2799 ★ |

Цены задаются через переменные окружения: `TARIFF_1M_RUB`, `TARIFF_3M_RUB`, `TARIFF_6M_RUB`, `TARIFF_12M_RUB` (и аналогичные для Stars). Значения по умолчанию указаны выше.

**Источник:** `bot/config.py`, `bot/utils/tariffs.py`

## Структура тарифов

Каждый тариф определяется в `BASE_TARIFF_SPECS`:

- **1 месяц** — базовый тариф для знакомства с сервисом
- **3 месяца** — средняя экономия, ~133 ₽/мес
- **6 месяцев** — выгодный вариант, ~125 ₽/мес
- **12 месяцев** — максимальная выгода, ~116 ₽/мес

Класс `Tariff` (dataclass) содержит: `code`, `title`, `duration_days`, `rub_price`, `stars_price`.

## Устройства

### Базовый лимит

- **По умолчанию:** 3 устройства на подписку
- **Максимум:** 8 устройств (с доп. слотами)

**Источник:** `bot/utils/device_slots.py` — `DEFAULT_DEVICE_LIMIT = 3`, `MAX_DEVICE_LIMIT = 8`

### Дополнительные слоты устройств

Пользователь может купить дополнительные слоты устройств:

- **Цена за слот:** 49 ₽ (переменная `DEVICE_SLOT_UNIT_PRICE_RUB`)
- **Максимум доп. слотов:** 5 (переменная `DEVICE_SLOT_MAX_EXTRA_SLOTS`)
- **Тип продукта:** `device_slot_addon`
- **Срок действия:** до конца текущей подписки (не фиксированный период)

Доп. слоты доступны только при активной платной подписке. Trial-пользователи не могут покупать слоты.

## Способы оплаты

| Способ | Тип | Описание |
|--------|-----|----------|
| Telegram Stars | Авто | Встроенная оплата Telegram |
| СБП (Platega) | Авто | Автоматическое подтверждение через провайдера Platega |
| СБП (ручная) | Ручная | Заявка с подтверждением администратора |
| Криптовалюта (Platega) | Авто | Через Platega, активы: USDT, TON |
| Криптовалюта (ручная) | Ручная | Реквизиты + проверка администратором |
| Баланс (RUB) | Авто | Внутренний баланс пользователя |

Время рассмотрения ручной заявки: по умолчанию **12 часов** (`MANUAL_PAYMENT_REVIEW_HOURS`).

**Источник:** `bot/config.py`, `bot/handlers/tariffs.py`

## Внутренний баланс

Модель `User` содержит поля:

- `balance_rub` — основной баланс (по умолчанию 0)
- `balance_reserved_rub` — зарезервированный баланс (для_pending_ оплат)

Баланс пополняется:
- Через реферальные бонусы
- Через ручное пополнение (balance topup)

При оплате списание с баланса происходит автоматически для RUB-методов (СБП, крипто). Telegram Stars оплачиваются отдельно, без смешивания с балансом.

Баланс может быть **положительным** (обычное состояние) или **нулевым**. Отрицательный баланс не предусмотрен — оплата с баланса невозможна при недостатке средств.

**Источник:** `backend/core/models.py` — `User.balance_rub`, `User.balance_reserved_rub`

## Автопродление

Автопродление как таковое **не реализовано** в текущей версии. Подписка не продлевается автоматически. Пользователь получает напоминания:

- За 3 дня до окончания подписки (тригер `subscription_expires_3d`)
- В день окончания подписки (триггер `subscription_expires_today`)

После окончания подписки доступ блокируется, VPN-ключи деактивируются. Для возобновления доступа нужно оплатить новый тариф.

**Источник:** `control_bot/storage.py` — `TRIGGER_DEFAULTS`

## Реферальная система

При первой оплате приглашённого пользователя:

- **Реферер** получает бонус на баланс
- **Приглашённый** тоже получает бонус

Бонусы зависят от тарифа:

| Тариф | Бонус рефереру | Бонус приглашённому |
|-------|---------------|---------------------|
| 1 месяц | 50 ₽ | 50 ₽ |
| 3 месяца | 50 ₽ | 50 ₽ |
| 6 месяцев | 50 ₽ | 50 ₽ |
| 12 месяцев | 100 ₽ | 100 ₽ |

**Источник:** `bot/utils/referrals.py` — `REFERRAL_BONUS_BY_TARIFF`

## Уровни реферальной программы

| Уровень | Оплаченных рефералов | Описание |
|---------|---------------------|----------|
| Без уровня | 0 | Начальное состояние |
| Новичок | 1–3 | До следующего: 4 всего |
| Продвинутый | 4–10 | До следующего: 11 всего |
| Партнер | 11+ | Максимальный уровень |

**Источник:** `bot/utils/referrals.py` — `REFERRAL_LEVELS`

## Промокоды

### Типы промокодов

| Тип | Код | Описание |
|-----|-----|----------|
| Скидка в процента | `discount_percent` | Процентная скидка (макс. 95%) |
| Дни доступа | `days_credit` | Бесплатные дни доступа |
| Подарочный | `gift_days` | Подарочная подписка (одноразовая) |

### Применение

- **Процентная скидка:** создаёт `pending_discount` редemption, применяется при следующей оплате
- **Дни доступа:** немедленно активирует подписку на указанное количество дней
- **Подарочный:** генерируется автоматически при покупке, одноразовый код

### Ограничения

- Максимум использований (`max_redemptions`) — по умолчанию 1
- Срок действия (`expires_at`) — опционально
- Один пользователь не может активировать один промокод дважды
- Нельзя иметь две активные процентные скидки одновременно

**Источник:** `backend/core/promo_codes.py`

## Тексты для реферальной системы

Настраиваются через переменные окружения:

- `REFERRAL_SHARE_TEXT` — текст при шеринге ссылки (по умолчанию: "Лучший сервис для доступа\nПереходи и получай бонусные рубли")
- `REFERRAL_REWARD_INVITED_TEXT` — текст для приглашённого при начислении бонуса
- `REFERRAL_REWARD_REFERRER_TEMPLATE` — шаблон для реферера (по умолчанию: "Ваш друг оплатил, вам начислено {bonus_rub} бонусных рублей")

**Источник:** `bot/config.py`
