# Политика конфиденциальности

**Amonora** (далее — «Сервис») обязуется защищать конфиденциальность пользователей. Настоящая Политика конфиденциальности описывает, какие данные собираются, как они используются и защищаются.

---

## 1. Сбор информации

### 1.1. Данные, предоставляемые пользователем

- **Telegram-идентификатор** (`user_id`, `username`) — при взаимодействии с ботами `@amonora_bot`, `@amonora_support_bot`, `@amonora_control_bot`
- **Платёжные данные** — информация, необходимая для обработки платежей через Telegram Stars, Platega (СБП/криптовалюты), а также ручной обработки платежей
- **Обращения в поддержку** — текст сообщений, медиафайлы (фото, видео, аудио), отправленные через support-бот
- **Промокоды и реферальные данные** — при активации промокодов или участии в реферальной программе

### 1.2. Данные, собираемые автоматически

- **HTTP-запросы** к `client.amonoraconnect.com` и `amonoraconnect.com`, включая:
  - User-Agent, заголовки браузеров
  - IP-адрес (для определения региона и предотвращения злоупотреблений)
- **Метаданные устройств** — тип платформы (Android, iOS, Windows, macOS, Linux, Apple TV, Android TV), информация о подключённых устройствах
- **Логи серверов** — журналы VPN-нод (Германия, Дания, Эстония) для мониторинга работоспособности и безопасности

### 1.3. Cookie-файлы

Информация о cookie-файлах содержится в отдельном документе: [Политика использования cookie](../08-вспомогательное/шаблоны/cookie-policy.md).

| Цель | Основание |
|------|-----------|
| Предоставление VPN-сервиса | Исполнение договора |
| Обработка платежей | Исполнение договора |

| Реферальная программа | Согласие пользователя |

| Безопасность и предотвращение злоупотреблений | Законный интерес |

---

## 3. Хранение и защита данных

- **Основное хранилище**: PostgreSQL — единый источник достоверных данных
- **Логи поддержки**: полная история обращений в БД без автоматического удаления
- **Доступ**: ограничен ролевой моделью (Владелец / Технический администратор / Менеджер)
- **Секреты и учётные данные**: не хранятся в репозитории; используются шаблоны `PRIVATE_ACCESS.template.md`

---

## 4. Передача данных третьим лицам

Сервис **не продаёт и не передаёт** персональные данные третьим лицам, за исключением:

- **Платёжные провайдеры** (Platega, Telegram Stars) — данные, необходимые для обработки транзакций
- **Требования законодательства** — по запросу уполномоченных органов в порядке, установленном законом
- **Инфраструктурные провайдеры** — хостинг VPN-нод (Германия, Дания, Эстония) — минимальные метаданные, необходимые для функционирования сети

---

## 5. Права пользователя

Пользователь вправе:

1. **Запросить** информацию о хранящихся персональных данных
2. **Потребовать исправления** неточных данных
3. **Потребовать удаления** данных (с учётом обязательств по хранению платёжных записей)
4. **Отозвать согласие** на обработку данных (с учётом последствий для работы сервиса)

Для реализации прав обратитесь через `@amonora_support_bot`.

---

## 6. Сроки хранения

| Тип данных | Срок хранения |
|------------|---------------|
| Аккаунт пользователя | До удаления аккаунта |
| Платёжные записи | В соответствии с применимым законодательством |
| Обращения в поддержку | Бессрочно (полная история в БД) |
| Аналитические события | ⚠️ Требуется уточнение |
| Логи VPN-нод | ⚠️ Требуется уточнение |
