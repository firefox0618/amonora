# Управление доступами

## Обзор

Этот документ описывает роли, доступы и процедуры управления доступами к инфраструктуре Amonora.

> **ВАЖНО**: Реальные пароли, ключи и токены хранятся в `PRIVATE_ACCESS.md` (НЕ в репозитории!). Для шаблонов используй `.env.example` и `*.template` файлы.

## Роли и доступы

| Роль | Серверы | БД | Боты | Панели | Команда |
|------|---------|----|------|--------|---------|
| **Owner** | root SSH (все) | postgres superuser | Все токены | Dashboard (owner), 3x-ui | @rudolf |
| **Tech** | root SSH (core, DK, EE) | postgres superuser | Все токены | Dashboard (tech), 3x-ui | @ruslan |
| **Support** | root SSH (core) | read-only | Support bot | Dashboard (support) | @muradym |
| **Control Admin** | — | — | Control bot | — | Настраивается через `control_admin_ids` |
| **Control Operator** | — | — | Control bot | — | Настраивается через `control_operator_ids` |

> **Примечание:** В текущем production-контуре отдельные роли доступа для внешних аналитических сервисов не используются.

## Dashboard роли

Определены в `.env.example`:

| Переменная | Username | Роль | Описание |
|-----------|----------|------|----------|
| `DASHBOARD_OWNER_USERNAME` | `rudolf` | Owner | Полный доступ, управление финансами, пользователями, серверами |
| `DASHBOARD_TECH_USERNAME` | `ruslan` | Tech | Технический доступ, серверы, VPN-конфигурация, ремонт |
| `DASHBOARD_SUPPORT_USERNAME` | `muradym` | Support | Поддержка, пользователи, билеты, ручные платежи |

Дашборд-сессии хранятся в БД, сессия истекает через:
- `DASHBOARD_SESSION_HOURS=24` — абсолютный TTL
- `DASHBOARD_SESSION_IDLE_MINUTES=45` — TTL неактивности

### Как выдавать SSH-доступ

1. Получить публичный ключ нового пользователя
2. Добавить в `/root/.ssh/authorized_keys` на нужном сервере
3. Добавить в `known_hosts` если нужно межсерверное взаимодействие
4. Обновить `PRIVATE_ACCESS.md` (локально, не в репозитории!)

### Как отзывать SSH-доступ

1. Удалить публичный ключ из `/root/.ssh/authorized_keys`
2. При компрометации — сгенерировать новые SSH-ключи для всех серверов

## Доступ к БД

### Пользователи PostgreSQL

| Пользователь | Права | Назначение |
|-------------|-------|-----------|
| `postgres` | Superuser | Системный пользователь |
| `amonora` | Full access to amonora_db | Приложение (боты, dashboard) |

> **Примечание:** В текущем production-контуре используется только PostgreSQL-пользователь `amonora` для приложений Amonora.

### Подключение

```bash
# С сервера core
psql -U amonora -d amonora_db

# БД только через localhost
psql -h 127.0.0.1 -U amonora -d amonora

# Как postgres
sudo -u postgres psql -d amonora_db
# Или через обёртку:
/opt/amonora_bot/ops/backup/psql_as_postgres.sh -d amonora_db
```

### Как выдавать доступ к БД

```sql
-- Создать нового пользователя
CREATE USER new_user WITH PASSWORD 'strong_password';

-- Дать read-only доступ
GRANT CONNECT ON DATABASE amonora_db TO new_user;
GRANT USAGE ON SCHEMA public TO new_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO new_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO new_user;

-- Дать полный доступ (для разработчика)
GRANT ALL PRIVILEGES ON DATABASE amonora_db TO new_user;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO new_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO new_user;
```

## VPN-панели (3x-ui)

### Доступ к панелям

| Панель | URL | Пользователь | Описание |
|--------|-----|-------------|----------|
| **Дания (3x-ui)** | Через туннель: `127.0.0.1:12053` | `XUI_USERNAME` / `XUI_PASSWORD` | 3x-ui Дания |
| **Эстония (3x-ui)** | Через туннель: `127.0.0.1:12054` | `XUI_USERNAME_EE` / `XUI_PASSWORD_EE` | 3x-ui Эстония |
| **Xray Core (Дания)** | Напрямую: `81.17.159.58` | Через SSH + config files | Xray Core, без панели |

### Переменные

```env
XUI_URL=http://127.0.0.1:12053
XUI_URL_DE=http://127.0.0.1:12053
XUI_URL_EE=http://127.0.0.1:12054/dashboard
XUI_USERNAME=
XUI_PASSWORD=
XUI_USERNAME_EE=
XUI_PASSWORD_EE=
```

### Как менять пароль 3x-ui

1. SSH к ноде
2. Открыть панель через туннель
3. Settings → изменить пароль
4. Обновить `.env` на core-сервере

> **Примечание:** Описаны только актуальные доступы к VPN-панелям и основным сервисам Amonora.

## Control Bot роли

Определяются через `config.py`:

| Роль | Переменная | Доступ |
|------|-----------|--------|
| Owner | `control_owner_ids` | Полный доступ |
| Admin | `control_admin_ids` | Административный доступ |
| Operator | `control_operator_ids` | Операционный доступ |
| Support (view-only) | `control_support_view_only_ids` | Только просмотр |

## Как выдавать доступ к Dashboard

1. Открыть Dashboard → Settings → Users
2. Добавить нового пользователя с ролью (owner/tech/support)
3. Установить пароль
4. Сообщить учётные данные через безопасный канал

## Как отзывать доступ

1. **Dashboard**: удалить пользователя в Settings
2. **SSH**: удалить публичный ключ из `authorized_keys`
3. **БД**: `DROP USER <username>;`
4. **Control Bot**: удалить ID из соответствующего списка в `.env`
5. **При компрометации**: сменить ВСЕ пароли и токены
