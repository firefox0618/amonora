# Модель угроз Amonora

## Активы (что защищаем)

| Актив | Описание | Критичность |
|-------|----------|------------|
| **Telegram-токены ботов** | Доступ к ботам (main, support, control, test) | Критический |
| **PostgreSQL БД** | Данные пользователей, платежи, аналитика, VPN-клиенты | Критический |
| **VPN-клиенты** | VLESS/Trojan ключи, UUID, email, expiry | Высокий |
| **Учётные данные панелей** | 3x-ui, Dashboard логины/пароли | Высокий |
| **SSH-ключи** | Доступ к серверам (core, DK, EE) | Критический |
| **Платёжные секреты** | Platega merchant_id/secret, CryptoPay API token | Высокий |
| **SSL-сертификаты** | TLS-сертификаты Let's Encrypt | Средний |
| **Конфигурации Xray** | Правила маршрутизации, whitelist IPs | Высокий |
| **Аналитические данные** | Воронки, источники, выручка | Средний |
| **OpenAI API Key** | Ключ для AI-функциональности | Низкий |

## Угрозы

### 1. Скомпрометирована нода

| Параметр | Описание |
|----------|----------|
| **Угроза** | Злоумышленник получил root-доступ к VPN-ноде (Дания/Эстония) |
| **Вероятность** | Средняя |
| **Воздействие** | Критическое: доступ к конфигам Xray, клиентским данным, возможность перехвата трафика |
| **Митигация** | 1. Single-IP enforcement (`xray_single_ip_enforcer.py`) ограничивает доступ с одного IP | 2. Whitelist IP (`amonora_dk_ip_whitelist.json`) | 3. Xray proxy protocol (xver=1) для идентификации | 4. Регулярные бэкапы конфигов (remote_artifact_backup) | 5. Мониторинг через server_watchdog |

**Действия при компрометации**:
1. Отозвать все VPN-клиенты на ноде
2. Сменить SSH-ключи
3. Пересоздать Xray config
4. Уведомить затронутых пользователей
5. Проверить логи на предмет несанкционированного доступа

### 2. Скомпрометирован токен бота

| Параметр | Описание |
|----------|----------|
| **Угроза** | Злоумышленник получил токен Telegram-бота |
| **Вероятность** | Средняя (утечка .env, лог с токеном, компрометация сервера) |
| **Воздействие** | Высокое: чтение/отправка сообщений от имени бота, доступ к данным через API |
| **Митигация** | 1. Токены только в .env (не в репозитории) | 2. `.gitignore` для .env | 3. ADMIN_IDS whitelist | 4. Регулярная ротация (каждые 90 дней) | 5. Мониторинг подозрительной активности |

**Действия при компрометации**:
1. НЕМЕДЛЕННО отозвать через BotFather
2. Обновить .env на сервере
3. Перезапустить сервис
4. Проверить логи на предмет несанкционированных действий

### 3. Утечка БД

| Параметр | Описание |
|----------|----------|
| **Угроза** | Утечка PostgreSQL с персональными и платёжными данными |
| **Вероятность** | Низкая (БД только на localhost, доступ через SSH с ключами) |
| **Воздействие** | Критическое: telegram_id, usernames, подписки, платёжная история, VPN-ключи |
| **Митигация** | 1. БД слушает только `127.0.0.1` | 2. Доступ только через SSH с ключами | 3. Бэкапы зашифрованы (gzip + SHA256) | 4. Offsite backup через rclone | 5. Еженедельный restore-validation drill |

**Хранящиеся данные**:
- `users`: telegram_id, username, subscription_status, trial/subscription expires_at, vpn_repair_needed, analytics source_key
- `vpn_clients`: protocol, email, client_uuid, metadata (country_code, inbound_id)
- `payment_records`: amount, provider, status, external_id, metadata
- `analytics_events`: event_name, user_id, source_key, source_mode
- `control_notification_events`: алерты и инциденты

### 4. DDoS

| Параметр | Описание |
|----------|----------|
| **Угроза** | DDoS-атака на публичные эндпоинты |
| **Вероятность** | Средняя |
| **Воздействие** | Высокое: недоступность сайта, бота, VPN |
| **Митигация** | 1. Cloudflare CDN (все домены через Cloudflare) | 2. `real_ip_header CF-Connecting-IP` в nginx | 3. Rate limiting в nginx: `limit_req_zone` для `/login`, `/bridge/access` | 4. `client_max_body_size 10m` | 5. Security headers (CSP, X-Frame-Options, HSTS) |

**Публичные эндпоинты под защитой**:

| Эндпоинт | Rate limit |
|----------|-----------|
| `/login` | 10r/m, burst=20 |
| `/verify` | 10r/m, burst=20 |
| `/bridge/access` | 1r/m, burst=2 |
| `/auth/` | 10r/m, burst=20 |

### 5. Несанкционированный доступ к панели

| Параметр | Описание |
|----------|----------|
| **Угроза** | Доступ к Dashboard или 3x-ui без авторизации |
| **Вероятность** | Низкая |
| **Воздействие** | Высокое: просмотр данных пользователей, изменение конфигов, финансовые операции |
| **Митигация** | 1. Dashboard: session-based auth, роли (owner/tech/support) | 2. 3x-ui: доступ только через SSH-туннель | 3. Cookie security: `DASHBOARD_COOKIE_SECURE=1` | 4. `limit_req` для login-эндпоинтов | 5. Audit log всех действий |

## Рекомендации

1. **Регулярная ротация**: токены каждые 90 дней, пароли — раз в 6 месяцев
2. **Мониторинг**: server_watchdog + control_error_triggers — непрерывный контроль
3. **Бэкапы**: ежедневные + еженедельный restore-validation drill
4. **Обновления**: регулярное обновление ОС, PostgreSQL, nginx, Python-зависимостей
5. **Логирование**: все действия через audit log, системные события через control_notification_events
6. **Разделение**: разные роли для разных уровней доступа
7. **НЕ хранить секреты в репозитории**: использовать `PRIVATE_ACCESS.md` вне git
