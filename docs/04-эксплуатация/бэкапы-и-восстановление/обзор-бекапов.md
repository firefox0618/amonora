# Бекапы и восстановление

## Архитектура бекапов

Бекапы Amonora работают по принципу **3-2-1**: 3 копии, 2 типа носителей, 1 вне сервера.

## Server-side бекапы

### PostgreSQL

| Параметр | Значение |
|----------|----------|
| Сервис | `amonora-core-pg-backup.service` + `.timer` |
| Формат | `pg_dump` (custom format) |
| Хранение | `/opt/amonora_bot/backups/` |
| Частота | Ежедневно |

### VPN-ноды (x-ui артефакты)

| Параметр | Значение |
|----------|----------|
| Сервис | `amonora-remote-artifact-backup@.service` + `.timer` |
| Что бекапится | `x-ui.db` (SQLite), конфиги 3x-ui |
| Хранение | `/opt/amonora_bot/backups/` |

## Off-host бекапы (Windows Task Scheduler)

Бекапы выгружаются с сервера на локальный ПК через **Windows Task Scheduler**:

| Скрипт | Язык | Назначение |
|--------|------|-----------|
| `backup_core_pg.ps1` | PowerShell | Скачивание дампа PostgreSQL с сервера |
| `backup_vpn_xui_artifacts.ps1` | PowerShell | Скачивание x-ui артефактов |
| `restore_core_pg_local.ps1` | PowerShell | Локальное восстановление из дампа |

**Путь на ПК:** `C:\Ops\Backups\amonora\`

### Как работает

1. PowerShell-скрипт подключается к серверу по SSH
2. Скачивает последние дампы
3. Проверяет целостность (SHA256)
4. Хранит по схеме retention (последние N бекапов)

## Retention

| Тип | Хранится |
|-----|----------|
| Server-side | Последние 3 бекапа (очистка через `cleanup.sh`) |
| Off-host | Настраивается в PowerShell-скриптах |
| Backup-архивы (`/opt/amonora_backup/`) | Последние 3 tar.gz (через `backup.sh`) |

## Restore-дрель

Результаты тестового восстановления:
- PostgreSQL дамп: 33K/886 строк, валидный
- Таблицы подтверждены: `users`, `vpn_clients`, `payment_records`, и др.
- `x-ui.db`: валидный SQLite
- ⚠️ Полный импорт в тестовый PostgreSQL **не верифицирован**

## Rollback

Скрипт `/opt/rollback.sh` — откат к предыдущей версии из backup.

## Быстрое восстановление PostgreSQL

```bash
# На сервере
sudo -u postgres psql -d amonora_db

# Проверка таблиц
\dt

# Проверка записей
SELECT count(*) FROM users;
SELECT count(*) FROM vpn_clients;
SELECT count(*) FROM payment_records;
```
