# Локальное восстановление

## Восстановление PostgreSQL

### Из бэкапа на сервере

```bash
ssh root@46.21.81.186

# 1. Найти последний бэкап
ls -lt /opt/amonora_bot/backups/core-pg/ | head -5
LATEST=$(ls -t /opt/amonora_bot/backups/core-pg/*/*.sql.gz | head -1)
echo "Бэкап: $LATEST"

# 2. Создать БД для восстановления (или использовать существующую)
# ВАЖНО: существующая БД будет перезаписана!

# 3. Восстановить
gunzip -c "$LATEST" | sudo -u postgres psql -d amonora

# Или через вспомогательные скрипты:
gunzip -c "$LATEST" | /opt/amonora_bot/ops/backup/psql_as_postgres.sh -d amonora
```

### На локальной машине (разработка)

```bash
# 1. Скопируй бэкап с сервера
scp root@46.21.81.186:/opt/amonora_bot/backups/core-pg/2026-04-14_09-00/amonora_db_2026-04-14_09-00.sql.gz ~/Downloads/

# 2. Распакуй
gunzip ~/Downloads/amonora_db_2026-04-14_09-00.sql.gz

# 3. Создай БД
createdb amonora_local

# 4. Восстанови
psql -d amonora_local -f ~/Downloads/amonora_db_2026-04-14_09-00.sql

# 5. Или используй готовый скрипт:
bash ops/local/restore_core_pg_local.sh
```

### Скрипт локального восстановления: `ops/local/restore_core_pg_local.sh`

```bash
#!/usr/bin/env bash
# Восстановление core PG на локальной машине
set -euo pipefail

BACKUP_FILE="${1:?Usage: restore_core_pg_local.sh <backup.sql.gz>}"
DB_NAME="${AMONORA_LOCAL_DB:-amonora_local}"

# Распаковка
TEMP_SQL=$(mktemp /tmp/amonora_restore_XXXXXX.sql)
trap 'rm -f "$TEMP_SQL"' EXIT

gunzip -c "$BACKUP_FILE" > "$TEMP_SQL"

# Создание БД
dropdb --if-exists "$DB_NAME"
createdb "$DB_NAME"

# Восстановление
psql -d "$DB_NAME" -f "$TEMP_SQL"

echo "Восстановлено в БД: $DB_NAME"
```

### PowerShell-скрипт (Windows): `ops/local/restore_core_pg_local.ps1`

```powershell
# Восстановление на Windows с установленным PostgreSQL
param(
    [Parameter(Mandatory)]
    [string]$BackupFile,

    [string]$DbName = "amonora_local"
)

# Распаковка
$TempSql = [System.IO.Path]::GetTempFileName() + ".sql"
& gunzip -c $BackupFile | Out-File -Encoding utf8 $TempSql

# Создание БД
& dropdb --if-exists $DbName
& createdb $DbName

# Восстановление
& psql -d $DbName -f $TempSql

Remove-Item $TempSql
Write-Host "Восстановлено в БД: $DbName"
```

## Восстановление на другом сервере

```bash
# 1. Скопируй бэкап на новый сервер
scp /opt/amonora_bot/backups/core-pg/2026-04-14_09-00/amonora_db_2026-04-14_09-00.sql.gz root@<new-server>:/tmp/

# 2. На новом сервере
ssh root@<new-server>
cd /tmp

# 3. Установи PostgreSQL (если нет)
apt update && apt install -y postgresql postgresql-contrib

# 4. Создай пользователя и БД
sudo -u postgres createuser --createdb amonora
sudo -u postgres createdb -O amonora amonora_db

# 5. Восстанови
gunzip -c amonora_db_2026-04-14_09-00.sql.gz | sudo -u postgres psql -d amonora_db
```

## Проверка после восстановления

```sql
-- 1. Проверить количество таблиц
SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = 'public';

-- 2. Проверить основные таблицы
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;

-- 3. Проверить количество записей
SELECT
    'users' AS table_name, COUNT(*) FROM users
UNION ALL
SELECT 'vpn_clients', COUNT(*) FROM vpn_clients
UNION ALL
SELECT 'payment_records', COUNT(*) FROM payment_records
UNION ALL
SELECT 'analytics_events', COUNT(*) FROM analytics_events;

-- 4. Проверить схемы
SELECT schema_name FROM information_schema.schemata ORDER BY schema_name;

-- 5. Проверить последовательности
SELECT sequence_name, last_value FROM information_schema.sequences
WHERE sequence_schema = 'public';
```

## Частые ошибки

### Ошибка: «database «amonora» already exists»

```bash
# Удалить существующую БД
dropdb amonora
# Или через вспомогательный скрипт
/opt/amonora_bot/ops/backup/dropdb_as_postgres.sh amonora
```

### Ошибка: «role «amonora» does not exist»

```bash
sudo -u postgres createuser --createdb amonora
```

### Ошибка: «permission denied for table ...»

```bash
# После восстановления проверить права
sudo -u postgres psql -d amonora_db -c "
  GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO amonora;
  GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO amonora;
"
```

### Ошибка: «pg_dump: server version mismatch»

Убедись, что версия pg_dump совпадает с версией PostgreSQL сервера:

```bash
pg_dump --version
psql --version
```

### Ошибка: восстановление заняло слишком много времени

- Проверь размер бэкапа: `ls -lh /path/to/backup.sql.gz`
- Проверь нагрузку на сервер: `top`, `iostat`
- Таймаут restore validation: 300 секунд (настраивается через `AMONORA_RESTORE_VALIDATION_TIMEOUT_SECONDS`)

### Ошибка: «invalid memory alloc request size»

Бэкап может быть повреён. Проверь SHA256:

```bash
sha256sum /opt/amonora_bot/backups/core-pg/*/amonora_db_*.sql.gz
cat /opt/amonora_bot/backups/status/core.json | grep artifact_sha256
```
