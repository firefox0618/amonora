# Стратегия бэкапов

## Обзор

Система бэкапов Amonora обеспечивает резервное копирование PostgreSQL, конфигураций VPN-нод и удалённых артефактов. Бэкапы выполняются автоматически через systemd-таймеры и проверяются еженедельно через restore-validation drill.

## Что бэкапится

| Объект | Метод | Путь |
|--------|-------|------|
| **PostgreSQL core** | `pg_dump` → gzip | `/opt/amonora_bot/backups/core-pg/` |
| **Конфигурации нод** | SSH → tar | `/opt/amonora_bot/backups/<source_key>/` |
| **Xray config (Дания)** | `remote_artifact_backup.sh` | tar.gz архив |
| **3x-ui config (Дания)** | через SSH туннель | — |
| **Статусы бэкапов** | JSON status files | `/opt/amonora_bot/backups/status/` |

## Расписание

| Задача | Таймер | Расписание | Persistent |
|--------|--------|------------|------------|
| PostgreSQL backup | `amonora-core-pg-backup.timer` | Ежедневно **09:00 UTC** | Да |
| Remote artifact backup | `amonora-remote-artifact-backup@.timer` | Ежедневно **09:30 UTC** | Да |
| Restore validation drill | `amonora-restore-validation.timer` | Воскресенье **05:10 UTC** | Да |

### Проверка таймеров

```bash
systemctl list-timers | grep amonora
```

## PostgreSQL бэкап

### Скрипт: `ops/backup/core_pg_backup.sh`

**Параметры** (через env):

| Переменная | Default | Описание |
|-----------|---------|----------|
| `AMONORA_BACKUP_ROOT` | `/opt/amonora_bot/backups` | Корневая директория бэкапов |
| `AMONORA_BACKUP_SOURCE_KEY` | `core` | Ключ источника |
| `AMONORA_PG_BACKUP_DIR` | `${BACKUP_ROOT}/core-pg` | Директория бэкапов БД |
| `AMONORA_PGDATABASE` | `amonora` | Имя БД |
| `AMONORA_PGUSER` | `amonora` | Пользователь БД |
| `AMONORA_PGHOST` | `127.0.0.1` | Хост БД |
| `AMONORA_PGPORT` | `5432` | Порт БД |
| `AMONORA_BACKUP_RETENTION_DAYS` | `7` | Хранение бэкапов |

**Процесс**:

1. `pg_dump` с параметрами: `--format=p --no-owner --no-privileges`
2. Gzip сжатие
3. SHA256 хеш артефакта
4. Offsite sync через rclone (если настроен)
5. Очистка старых бэкапов (> 7 дней)
6. Запись статуса в JSON

**Формат артефакта**:
```
/opt/amonora_bot/backups/core-pg/2026-04-14_09-00/amonora_db_2026-04-14_09-00.sql.gz
```

### Сервис: `amonora-core-pg-backup.service`

```ini
[Unit]
After=network-online.target postgresql.service

[Service]
Type=oneshot
EnvironmentFile=-/etc/amonora/backup.env
ExecStart=/bin/bash /opt/amonora_bot/ops/backup/core_pg_backup.sh
```

## Remote artifact backup

### Скрипт: `ops/backup/remote_artifact_backup.sh`

Бэкапит конфигурации с удалённых серверов через SSH + tar:

**Параметры**:

| Переменная | Обязательная | Описание |
|-----------|-------------|----------|
| `AMONORA_REMOTE_SOURCE_KEY` | Да | Ключ (например, `dk-xray-config`) |
| `AMONORA_REMOTE_HOST` | Да | IP удалённого сервера |
| `AMONORA_REMOTE_PATH` | Да | Путь к файлу/директории на удалённом сервере |
| `AMONORA_REMOTE_SSH_USER` | Нет (root) | SSH пользователь |
| `AMONORA_REMOTE_SSH_PORT` | Нет (22) | SSH порт |
| `AMONORA_REMOTE_SSH_KEY` | Нет | Путь к SSH ключу |

**Процесс**:
1. SSH к удалённому серверу
2. `tar -czf -` на удалённой машине
3. Сохранение локально
4. SHA256 хеш
5. Offsite sync через rclone
6. Очистка старых бэкапов

### Сервис: `amonora-remote-artifact-backup@.service`

Шаблонный сервис (`%i` — instance name):

```ini
[Service]
Type=oneshot
EnvironmentFile=-/etc/amonora/backup.env
EnvironmentFile=/etc/amonora/backup.d/%i.env
ExecStart=/bin/bash /opt/amonora_bot/ops/backup/remote_artifact_backup.sh
```

Каждый instance имеет свой env-файл в `/etc/amonora/backup.d/<name>.env`.

## Проверка валидности

### Скрипт: `ops/backup/restore_validation_check.sh`

Еженедельный drill восстановления:

**Процесс**:
1. Находит последний `.sql.gz` бэкап
2. Создаёт временную БД: `amonora_restore_probe_YYYYMMDDHHmmss`
3. Восстанавливает бэкап: `gunzip -c | psql --set ON_ERROR_STOP=1`
4. Проверяет: `SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = 'public'`
5. Если таблиц > 0 — успех
6. Удаляет временную БД (cleanup trap)

**Таймаут**: 300 секунд (5 минут)

**Результат** сохраняется в:
- `/opt/amonora_bot/backups/status/restore-validation.json`
- `/opt/amonora_bot/backups/status/restore-proof.json`

### Сервис: `amonora-restore-validation.service`

```ini
[Unit]
After=network-online.target postgresql.service amonora-core-pg-backup.service

[Service]
Type=oneshot
EnvironmentFile=-/etc/amonora/backup.env
ExecStart=/bin/bash /opt/amonora_bot/ops/backup/restore_validation_check.sh
```

## Remote backup (offsite)

Если настроен rclone, бэкапы синхронизируются в облачное хранилище:

| Переменная | Описание |
|-----------|----------|
| `AMONORA_BACKUP_RCLONE_CMD` | Путь к rclone (по умолчанию `rclone`) |
| `AMONORA_BACKUP_RCLONE_REMOTE` | Имя rclone remote (пусто = offsite отключён) |

**Структура на удалённом хранилище**:
```
<rclone-remote>/core-pg/2026-04-14_09-00/
<rclone-remote>/dk-xray-config/2026-04-14_09-30/
```

## Таблица стратегий

| Стратегия | Что | Когда | Хранение | Проверка |
|-----------|-----|-------|----------|----------|
| PostgreSQL dump | Вся БД amonora | Ежедневно 09:00 UTC | 7 дней | Еженедельный restore drill |
| Remote artifacts | Конфиги нод через SSH | Ежедневно 09:30 UTC | 7 дней | SHA256 + rclone |
| Restore validation | Проверка восстанавливаемости | Воскресенье 05:10 UTC | — | COUNT таблиц > 0 |

## Проверка статуса бэкапов

```bash
# Статус последнего PG бэкапа
cat /opt/amonora_bot/backups/status/core.json

# Статус restore validation
cat /opt/amonora_bot/backups/status/restore-validation.json
cat /opt/amonora_bot/backups/status/restore-proof.json

# Список бэкапов
ls -la /opt/amonora_bot/backups/core-pg/

# Размер бэкапов
du -sh /opt/amonora_bot/backups/
```

### Пример статуса core.json

```json
{
  "source_key": "core",
  "runner": "server-side",
  "last_backup_at": "2026-04-14T09:00:00Z",

  "artifact_sha256": "abc123...",
  "offsite_status": "synced",
  "offsite_remote": "backblaze-b2",
  "offsite_synced_at": "2026-04-14T09:01:30Z"
}
```

## Вспомогательные скрипты

| Скрипт | Назначение |
|--------|-----------|
| `ops/backup/psql_as_postgres.sh` | Обёртка `sudo -u postgres psql` |
| `ops/backup/pg_dump_as_postgres.sh` | Обёртка `sudo -u postgres pg_dump` |
| `ops/backup/createdb_as_postgres.sh` | Обёртка `sudo -u postgres createdb` |
| `ops/backup/dropdb_as_postgres.sh` | Обёртка `sudo -u postgres dropdb` |

Эти скрипты обеспечивают выполнение команд от имени пользователя postgres без прямого входа в систему.
