# Развёртывание Amonora

## Предварительные проверки перед деплоем

Перед выполнением деплоя убедись, что:

1. **Доступ к серверу**: SSH-доступ к `amonora-core` (IP `46.21.81.186`) работает.
2. **PostgreSQL запущен**: `systemctl is-active postgresql` возвращает `active`.
3. **Nginx запущен**: `systemctl is-active nginx` возвращает `active`.
4. **Все сервисы активны**: проверь через `systemctl list-units 'amonora-*' --state=active`.
5. **Бэкап БД существует**: проверь `/opt/amonora_bot/backups/core-pg/` — последний бэкап не старше 24 часов.
6. **Нет активных инцидентов**: проверь дашборд

## Шаги деплоя

### 1. Обновление кода

```bash
ssh root@46.21.81.186
cd /opt/amonora_bot
git pull origin develop
```

### 2. Установка зависимостей

```bash
# Python-зависимости (для ботов и ops-скриптов)
/opt/amonora_bot/venv/bin/pip install -r requirements.txt

# Dashboard UI (если изменились package.json)
cd /opt/amonora_bot/dashboard/ui
npm install && npm run build
```

### 3. Проверка конфигурации

```bash
# Убедись, что .env актуален
cat /opt/amonora_bot/.env | grep -v PASSWORD | grep -v TOKEN | grep -v SECRET

# Если добавлены новые переменные — обнови .env
# ВАЖНО: пароли и токены в .env, НЕ в репозитории!
```

### 4. Перезапуск сервисов

```bash
# Сначала БД (если была миграция схемы)
systemctl restart postgresql

# Затем — бэкап-система
systemctl restart amonora-xui-tunnel

# Основные сервисы (порядок важен!)
systemctl restart amonora-control-bot
systemctl restart amonora-dashboard-ui
systemctl restart amonora-test-bot
systemctl restart nginx

# Фоновые таймеры перезапускать не нужно — они подхватят изменения сами
# Но можно принудительно запустить:
systemctl restart amonora-server-watchdog
systemctl restart amonora-analytics-refresh
```

## Rollback процедура

Если после деплоя что-то пошло не так:

```bash
ssh root@46.21.81.186
cd /opt/amonora_bot

# 1. Откат к предыдущему коммиту
git log --oneline -10  # найди нужный коммит
git reset --hard <PREV_COMMIT_HASH>

# 2. Перезапусти сервисы
systemctl restart amonora-control-bot
systemctl restart amonora-dashboard-ui
systemctl restart amonora-test-bot
systemctl restart nginx

# 3. Если проблема в БД — восстанови из бэкапа
# Смотри: docs/04-эксплуатация/бэкапы-и-восстановление/локальное-восстановление.md
```
