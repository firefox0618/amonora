# Обновление SSL-сертификатов

## Обзор

Все публичные сервисы Amonora работают через HTTPS с сертификатами Let's Encrypt, управляемыми Certbot. Сертификаты установлены на nginx.

## Домены с SSL

| Домен | Сертификат | Назначение | Nginx конфиг |
|-------|-----------|-----------|-------------|
| `amonoraconnect.com` | `/etc/letsencrypt/live/amonoraconnect.com/` | Основной сайт, Dashboard, Landing | `amonora-dashboard.server.conf` |
| `www.amonoraconnect.com` | `/etc/letsencrypt/live/amonoraconnect.com/` | Канонический вход | `amonora-dashboard.server.conf` |
| `client.amonoraconnect.com` | `/etc/letsencrypt/live/client.amonoraconnect.com/` | Subscription page, Happ wrapper | `amonora-client.server.conf` |
| `est.amonoraconnect.com` | `/etc/letsencrypt/live/amonoraconnect.com/` | Estonia web-edge | `amonora-estonia-web-edge.server.conf` |

> **Примечание:** Сертификат `grafana.amonoraconnect.com` больше не используется — Grafana удалена.

## SSL-конфигурация

Во всех nginx-конфигах применяется единая конфигурация:

```nginx
ssl_protocols TLSv1.2 TLSv1.3;
ssl_session_timeout 1d;
ssl_session_cache shared:SSL:10m;
ssl_session_tickets off;

# HSTS
add_header Strict-Transport-Security "max-age=31536000" always;

# Security headers
add_header X-Frame-Options "DENY" always;
add_header X-Content-Type-Options "nosniff" always;
add_header Referrer-Policy "no-referrer" always;
add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;
add_header Content-Security-Policy "default-src 'self'; ..." always;
```

## Автоматическое продление

Certbot устанавливает systemd timer для автоматического продления:

```bash
# Проверить статус timer
systemctl status certbot.timer

# Проверить следующее выполнение
systemctl list-timers certbot.timer

# Принудительный dry-run
certbot renew --dry-run
```

### Расписание

Certbot по умолчанию проверяет сертификаты **дважды в день** и обновляет их, если до истечения срока осталось менее 30 дней.

## Ручное продление

### Стандартное продление

```bash
# Продлить все сертификаты
certbot renew

# Перезапустить nginx для применения
systemctl reload nginx
```

### Продление конкретного домена

```bash
certbot certonly --nginx -d amonoraconnect.com -d www.amonoraconnect.com
certbot certonly --nginx -d client.amonoraconnect.com
```

Для `est.amonoraconnect.com` на сервере Эстонии:

```bash
ssh root@<estonia-host>
certbot renew
systemctl reload nginx
```

Для `amonoraconnect.com` на Estonia web-edge — аналогично:

```bash
ssh root@<estonia-host>
certbot certonly --nginx -d amonoraconnect.com -d www.amonoraconnect.com
systemctl reload nginx
```

## Проверка после обновления

### 1. Проверить nginx

```bash
nginx -t
systemctl reload nginx
```

### 2. Проверить срок действия сертификата

```bash
# Проверить конкретный домен
echo | openssl s_client -servername amonoraconnect.com -connect amonoraconnect.com:443 2>/dev/null | openssl x509 -noout -dates

# Или через certbot
certbot certificates
```

### 3. Проверить HTTPS в браузере

Открой `https://amonoraconnect.com/` и проверь замок в адресной строке.

### 4. Проверить API-эндпоинты

```bash
curl -sI https://amonoraconnect.com/health | grep HTTP
curl -sI https://client.amonoraconnect.com/ | grep HTTP
```

## Cloudflare

Все домены работают через Cloudflare (видно по `CF-Connecting-IP` и `set_real_ip_from` в nginx-конфигах). Cloudflare также предоставляет свой TLS-сертификат для edge-шифрования.

Рекомендуется:

- **SSL/TLS mode в Cloudflare**: Full (strict)
- **Edge certificate**: Let's Encrypt или Universal SSL

## Расписание

| Задача | Когда |
|--------|-------|
| Автоматическое продление | 2 раза в день (certbot timer) |
| Ручная проверка | Раз в месяц |
| Проверка логов | После каждого продления |
| Проверка Cloudflare SSL | При изменениях в DNS |

## Частые ошибки

### Ошибка: «Failed to renew certificate»

```bash
# Проверить DNS
dig amonoraconnect.com +short

# Проверить nginx config
nginx -t

# Проверить логи certbot
cat /var/log/letsencrypt/letsencrypt.log
```

### Ошибка: «Too many certificates already issued»

Подожди 168 часов (1 неделю) — лимит Let's Encrypt на домен.

### Ошибка: «Certificate not yet valid»

Проверь время на сервере:

```bash
date
timedatectl
```
