# Ротация токенов

## Обзор

Система Amonora использует несколько Telegram-ботов, каждый со своим токеном. Токены хранятся в `.env` файле на сервере и передаются через переменные окружения.

## Таблица ботов

| Бот | Переменная .env | Systemd сервис | Назначение |
|-----|----------------|---------------|-----------|
| **Main Bot** (`@amonora_bot`) | `BOT_TOKEN` | — (работает через backend) | Клиентский бот — основной вход |
| **Support Bot** (`@amonora_support_bot`) | `SUPPORT_BOT_TOKEN` | — | Бот поддержки |
| **Control Bot** (`@amonora_control_bot`) | `AMONORA_CONTROL_BOT_TOKEN` | `amonora-control-bot.service` | Админ-панель в Telegram |
| **Test Bot** (`@amonora_v_2_0_bot`) | `AMONORA_TEST_BOT_TOKEN` | `amonora-test-bot.service` | Тестовый бот для разработки |

### Дополнительные переменные

| Переменная | Описание |
|-----------|----------|
| `ADMIN_IDS` | ID администраторов основного бота |
| `SUPPORT_ADMIN_IDS` | ID администраторов support-бота |
| `AMONORA_INTERNAL_CHANNEL_WEBHOOK_SECRET` | Секрет вебхука канала |
| `CRYPTO_PAY_API_TOKEN` | Токен CryptoPay |
| `OPENAI_API_KEY` | API-ключ OpenAI |

## Процедура замены токена

### Плановая ротация (каждые 90 дней)

#### Шаг 1: Получить новый токен

1. Открой BotFather в Telegram: `@BotFather`
2. Отправь `/mybots`
3. Выбери нужного бота
4. Нажми `Bot Settings` → `Token`
5. Нажми `Revoke current token`
6. Скопируй новый токен

#### Шаг 2: Обновить .env на сервере

```bash
ssh root@46.21.81.186

# Открой .env
nano /opt/amonora_bot/.env

# Замени токен (пример для Control Bot):
# Было: AMONORA_CONTROL_BOT_TOKEN=123456:OLD-TOKEN-HERE
# Стало: AMONORA_CONTROL_BOT_TOKEN=123456:NEW-TOKEN-HERE
```

#### Шаг 3: Перезапустить сервис

```bash
# Для Control Bot
systemctl restart amonora-control-bot

# Для Test Bot
systemctl restart amonora-test-bot

# Для Main Bot (через backend) — перезапустить backend
# (зависит от конфигурации deployment)
```

#### Шаг 4: Проверить

```bash
# Проверить логи
journalctl -u amonora-control-bot -n 30 --no-pager --since "2 min ago"

# Проверить что бот отвечает
# Открой @amonora_control_bot → /start
```

### Экстренная ротация (при компрометации)

Если токен утечёт или будет скомпрометирован:

```bash
# 1. НЕМЕДЛЕННО отозвать токен через BotFather
#    @BotFather → /mybots → выбрать бота → Bot Settings → Token → Revoke

# 2. Быстро обновить .env
ssh root@46.21.81.186
# Заменить токен в /opt/amonora_bot/.env

# 3. Перезапустить ВСЕ зависимые сервисы
systemctl restart amonora-control-bot
systemctl restart amonora-test-bot
# Перезапустить backend (main bot)

# 4. Проверить логи на предмет несанкционированного использования
journalctl -u amonora-control-bot -n 500 --no-pager --since "24 hours ago"

# 5. Проверить dashboard audit log
# Dashboard → /audit — посмотреть подозрительные действия

# 6. Если компрометация через утечку .env — сменить ВСЕ токены и секреты
```

## Проверка после замены

### 1. Сервис запущен

```bash
systemctl status amonora-control-bot
systemctl status amonora-test-bot
```

### 2. Бот отвечает

```bash
# В Telegram: отправить /start нужному боту
# Должен ответить без ошибок
```

### 3. Логи чистые

```bash
journalctl -u amonora-control-bot -n 50 --no-pager | grep -i "error\|forbidden\|unauthorized"
```

### 4. Control-бот команды работают

```bash
# В @amonora_control_bot:
# /start — должен открыть меню
# /status — должен показать статус
# /alerts — должен показать алерты
```

## Безопасность хранения токенов

- **НЕ коммить** `.env` в репозиторий
- **НЕ передавать** токены через мессенджеры
- **Использовать** `PRIVATE_ACCESS.template.md` для шаблонов
- Хранить `.env` с правами `600`:

```bash
chmod 600 /opt/amonora_bot/.env
chown root:root /opt/amonora_bot/.env
```

## Зависимости

Токены используются в следующих модулях:

| Модуль | Какие токены использует |
|--------|----------------------|
| `bot/config.py` | Все токены через `load_dotenv()` |
| `control_bot/main.py` | `config.control_bot_token` |
| `ops/access_reminders.py` | `config.bot_token`, `config.support_bot_token` |
| `ops/server_watchdog.py` | `config.bot_token` или `config.support_bot_token` |
| `bot/vpn_api.py` | `config.xui_username`, `config.xui_password` |
| `bot/platega.py` | `config.platega_merchant_id`, `config.platega_secret_key` |
