# Amonora Control — Финальное ТЗ v4

## 0. Концепция

Старая панель полностью удаляется.  
Создаётся новая система с нуля — как операционный центр управления Amonora Connect.

Главная цель:
> Деньги → доступ → работает  
Если не работает → 1 кнопка должна исправить.

---

## 1. Роли

### Владелец
- полный доступ ко всему

### Технический админ
- подтверждает платежи
- управляет пользователями
- управляет серверами
- делает sync / repair
- может всё кроме управления владельцем

### Менеджер
- подтверждает платежи
- работает с пользователями
- работает с поддержкой

---

## 2. Пользователи

### Статусы
- активен
- пробный
- без доступа
- заблокирован
- ожидает оплату
- ошибка синхронизации
- требует ремонта

### Устройства
- максимум 3 устройства
- 4-е может создать только админ
- 1 ключ = 1 устройство
- удаление устройства = полное удаление
- при перевыпуске старый ключ перестает работать

### Данные устройства
- тип
- IP
- регион
- нода
- last seen

---

## 3. Подписка и доступ

- доступ пропадает при окончании
- контроль на сервере обязателен
- продление добавляется к текущему сроку
- баланс хранится и отображается

---

## 4. Sync / Repair (КЛЮЧЕВОЕ)

### Кнопка "Синхронизировать"
Должна:
- проверить доступ
- проверить тариф
- проверить устройства
- пересоздать ключ
- перепривязать ноду
- применить доступ
- восстановить работу

### Кнопка "Глубокий ремонт"
- полная пересборка состояния

### Если не помогло
- статус: требует ремонта
- алерт админу
- запись в лог

---

## 5. Платежи

### Методы
- Telegram Stars
- СБП (ручной)
- крипта (ручная)

### Статусы
- ожидает оплату
- ожидает подтверждения
- подтвержден
- отклонен
- истек
- спорный
- ошибка

### Flow
1. пользователь платит
2. нажимает "я оплатил"
3. приходит в control bot
4. админ подтверждает

### После подтверждения
- выдать доступ
- продлить
- sync
- уведомить пользователя
