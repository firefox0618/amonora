# Панель управления Amonora

> Supporting reference document. Use [ARCHITECTURE.md](/home/dextrmed/projects/amonora_bot/documentation/ARCHITECTURE.md), [FEATURES.md](/home/dextrmed/projects/amonora_bot/documentation/FEATURES.md) and [RUNBOOK.md](/home/dextrmed/projects/amonora_bot/documentation/RUNBOOK.md) as canonical current-state docs first.

## Назначение

`Панель управления Amonora` — внутренняя веб-панель команды.

Подробная техническая карта нового Next.js-контура вынесена отдельно:

- `panel-ui-deep-dive.md`

Она нужна, чтобы в одном месте управлять:

- пользователями
- VPN-устройствами
- поддержкой
- платежами
- серверами
- сервисами
- документацией

## Как войти

1. Открой `https://amonoraconnect.com/login`
2. Введи логин и пароль администратора
3. Получи код в `@amonora_control_bot`
4. Подтверди вход на экране `verify`

Важно:

- код приходит только в `Amonora Control`
- если бот ни разу не был открыт, сначала нужно нажать `/start`

## Как пользоваться основными разделами

### Обзор

Используй `Обзор`, чтобы быстро увидеть:

- общее состояние системы
- проблемные платежи
- support-нагрузку
- статусы нод и сервисов

### Пользователи

В разделе `Пользователи` можно:

- открыть карточку пользователя
- посмотреть доступ и устройства
- создать или удалить устройство
- выдать trial
- продлить или снять доступ
- запустить repair / re-sync VPN

### Платежи

В разделе `Платежи` можно:

- смотреть ручные заявки
- подтверждать и отклонять платежи
- видеть, сколько списано с Баланса
- открывать связанного пользователя

### Поддержка

В разделе `Поддержка` можно:

- открыть очередь тикетов
- читать историю диалога
- смотреть клиентские фото / видео / аудио
- отвечать пользователю
- назначать и закрывать обращения

### Серверы

В разделе `Серверы` можно:

- видеть статусы нод
- проверять загрузку и задержки
- смотреть проблемные managed servers

### Сервисы

В разделе `Сервисы` можно:

- смотреть systemd-статусы
- менять RUB-тарифы
- проверять env и operational runtime

### Документация

В разделе `Документация` можно:

- открыть knowledge-материалы
- посмотреть rollout reports
- быстро перейти к runbook и supporting docs

## Что показывает вкладка Документация

Новый раздел документации нужен как встроенная база знаний.

Он должен:

- читать список документов из GitHub
- показывать структуру документации прямо в панели
- красиво рендерить Markdown
- иметь fallback на локальную копию, если GitHub недоступен

## Что важно по дизайну

Панель должна ощущаться как продукт, а не как набор форм:

- сильная типографика
- читаемые карточки
- понятные статусы
- акцентные цвета для предупреждений
- чистая навигация без мусора

## Серверный раздел

Во вкладке `Серверы` важно видеть:

- нагрузку
- цветовые предупреждения
- ping
- скорость
- статусы нод

## Сервисный раздел

Во вкладке `Сервисы` важно:

- видеть systemd-статусы
- перезапускать сервисы
- смотреть логи
- править `.env`
- менять тарифы

## Принцип развития

Панель управления должна быть единым центром управления всей экосистемой `Amonora`, а не только VPN-ботом.
