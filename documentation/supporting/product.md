# Amonora: продукт и бренд

> Supporting product/brand reference. Use [PROJECT_OVERVIEW.md](/home/dextrmed/projects/amonora_bot/documentation/PROJECT_OVERVIEW.md) and [FEATURES.md](/home/dextrmed/projects/amonora_bot/documentation/FEATURES.md) as canonical current-state docs first.

## Что такое Amonora

`Amonora` — это не только VPN-бот, а базовый бренд для экосистемы сервисов.

На текущем этапе главное направление — `Amonora`, но сам бренд строится шире:

- инфраструктурные сервисы
- digital-продукты
- поддержка и автоматизация
- внутренние панели управления

## Текущий продукт

Сейчас активный пользовательский продукт — Telegram-бот для выдачи VPN-доступа.

Основные сценарии:

- проверка доступа
- пробный доступ
- платная подписка
- создание устройств
- выбор протокола
- выбор страны
- получение ключей и конфигов
- поддержка через отдельного support-бота

## Как должен ощущаться бренд

По тону и визуалу `Amonora` — это:

- технологично
- премиально
- спокойно
- современно
- без случайных визуальных решений

Важно, чтобы интерфейсы выглядели как единая система:

- бот
- dashboard
- документация
- support

## Продуктовые опоры

### 1. Простота

Пользователь должен пройти путь:

`зашёл -> получил доступ -> создал устройство -> подключился`

без лишней нагрузки и без необходимости разбираться в технических деталях.

### 2. Контроль

Команда должна видеть:

- пользователей
- устройства
- серверы
- поддержку
- платежи

в одном месте.

### 3. Масштабируемость

Архитектура проекта должна позволять:

- перенос backend на отдельный сервер
- добавление новых VPN-нод
- рост support-процессов
- развитие dashboard в полноценный центр управления

## Брендовая стратегия на текущий этап

На сегодня рабочая структура такая:

- `Amonora` — публичный сервис для клиентов
- `Amonora Control Center` — внутренняя веб-панель команды
- `Amonora Support Bot` — отдельная линия поддержки

## Что не должно происходить

- смешивание backend и heavy VPN-нагрузки на одном слабом VPS
- хранение проектных знаний только в Telegram и устных договорённостях
- расхождение между тем, что реально работает, и тем, что написано в документации
